# opencloud-monitor 应用设计文档

---

## 1. 模块定位

opencloud-monitor 是 opencloud-console 的**库模块（jar，无独立启动类）**，由 `opencloud-web` 引入并随主应用启动。模块共享同一 JVM、数据库连接池（PostgreSQL）和 Redis 连接。

鉴权统一由 `opencloud-auth` 模块的 `JwtAuthenticationFilter` 处理，monitor 模块只需通过 `@PreAuthorize` 做接口级权限控制。

## 3. 对外 API 接口

### 3.1 指标查询（Metrics）

| 接口 | 方法 | 路径 | 权限 | 描述 |
|------|------|------|------|------|
| 即时查询 | GET | `/api/monitor/metrics/query` | `monitor:metrics:query` | PromQL 即时查询 |
| 范围查询 | GET | `/api/monitor/metrics/query_range` | `monitor:metrics:query` | PromQL 时序数据 |
| 集群总览 | GET | `/api/monitor/metrics/overview` | `monitor:metrics:query` | 聚合 CPU/内存/磁盘/网络 |
| 节点指标 | GET | `/api/monitor/metrics/nodes` | `monitor:metrics:query` | 各节点资源使用率 |
| 指标名称列表 | GET | `/api/monitor/metrics/labels` | `monitor:metrics:query` | 可用指标名称（PromQL 补全） |

**Request/Response 示例（即时查询）：**

```
GET /api/monitor/metrics/query?query=up&time=1711900000

Response:
{
  "code": 200,
  "data": {
    "status": "success",
    "data": {
      "resultType": "vector",
      "result": [...]
    }
  }
}
```

**Request/Response 示例（范围查询）：**

```
GET /api/monitor/metrics/query_range?query=rate(node_cpu_seconds_total[5m])&start=1711890000&end=1711900000&step=60

Response: 透传 Prometheus 原始响应结构
```

## 4. 核心组件设计

### 4.1 PrometheusClient

```java
@Component
public class PrometheusClient {
    // 基于 WebClient，配置超时 10s
    // query()    → GET /api/v1/query
    // queryRange() → GET /api/v1/query_range
    // labelValues() → GET /api/v1/label/__name__/values
    // 统一处理 Prometheus 返回的 status=error 情况
    // 抛出 MonitorException.prometheusQueryError()
}
```

### 4.2 GrafanaClient

```java
@Component
public class GrafanaClient {
    // 基于 WebClient，携带 Bearer Token
    // searchDashboards() → GET /api/search
    // buildEmbedUrl(uid, from, to) → 构建带参数的 kiosk URL
}
```

### 4.3 NotifySender 策略模式

```java
public interface NotifySender {
    ChannelType supportType();
    void send(NotifyContext context) throws NotifyException;
}

// NotifyContext 包含：
// - MonitorNotifyChannel channel（已解密）
// - MonitorAlertRecord record
// - boolean isTest
// - String testMessage（测试时使用）
```

| 实现类 | 渠道类型 | 核心依赖 |
|--------|---------|---------|
| `EmailNotifySender` | EMAIL | Spring Boot Mail（`JavaMailSender`） |
| `DingTalkNotifySender` | DINGTALK | WebClient，HMAC-SHA256 加签 |
| `WeChatWorkNotifySender` | WECHAT_WORK | WebClient |
| `WebhookNotifySender` | WEBHOOK | WebClient，Freemarker 模板渲染 |

### 4.4 WebSocketSessionRegistry

```java
@Component
public class WebSocketSessionRegistry {
    // ConcurrentHashMap<String userId, WebSocketSession session>
    // register(userId, session)    → 同一用户新连接替换旧连接并关闭旧 Session
    // unregister(userId)
    // broadcast(String message)    → 遍历所有活跃 Session 发送消息
    // sendToUser(userId, message)  → 向指定用户发送
    // getActiveSessions()          → 返回当前活跃连接数
}
```

### 4.5 AlertWebSocketHandler

```java
@Component
public class AlertWebSocketHandler extends TextWebSocketHandler {
    // afterConnectionEstablished → 从 URI query 解析 token → 校验 → register
    // handleTextMessage → 处理 PONG 心跳
    // afterConnectionClosed → unregister
    // handleTransportError → 记录日志，unregister
}
```

### 4.6 AlertWebhookServiceImpl

```java
// 处理 AlertManager 推送，核心逻辑：
// 1. 签名校验（可选）
// 2. 遍历 alerts 列表
// 3. 幂等写入（by alertName + instance + firedAt）
// 4. 检查静默规则
// 5. 触发 AlertPushService.broadcast()（同步）
// 6. @Async 发送多渠道通知 + 写 monitor_notify_log
// 7. 恢复通知处理（resolved 状态）
```

### 4.7 加密工具（AES-256-GCM）

```java
// 位置：com.qoobot.opencloud.monitor.util.AesEncryptUtil
// encryptField(plaintext)  → Base64(GCM_IV + ciphertext)
```

## 5. 配置项

> 配置统一放在 `opencloud-web/src/main/resources/application.yml`

```yaml
opencloud:
  prometheus:
    url: ${PROMETHEUS_URL:http://prometheus:9090}
    timeout: 10000           # 查询超时（ms）
    max-range-days: 30       # 范围查询最大天数

  grafana:
    url: ${GRAFANA_URL:http://grafana:3000}
    token: ${GRAFANA_TOKEN:}
    kiosk: true              # 嵌入时使用 kiosk 模式

  monitor:
    encrypt-key: ${MONITOR_ENCRYPT_KEY:default-32bytes-key!!!!!!!}  # 通知渠道敏感字段加密密钥
    webhook-secret: ${ALERT_WEBHOOK_SECRET:}                        # AlertManager Webhook 签名密钥（空=不校验）
    notify-async-pool-size: 5  # 通知发送异步线程池大小
    notify-retry-times: 3      # 通知发送最大重试次数
```
