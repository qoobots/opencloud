## 一、项目定位与目标

| **业务层** | 提供类阿里云控制台的用户体验 | 功能完整度 ≥ 80% |

### 2.2 核心业务流程

#### 2.2.1 用户认证流程

```
┌─────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  用户    │────▶│  登录请求   │────▶│  身份验证   │────▶│  生成 Token │
│         │     │  /login     │     │  用户名/密码 │     │  JWT+Refresh│
└─────────┘     └─────────────┘     └─────────────┘     └──────┬──────┘
                                                                │
                                                                ▼
┌─────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  用户    │◀────│  返回 Token │◀────│  Redis 存储 │◀────│  权限加载   │
│         │     │  Access/    │     │  会话信息   │     │  角色/权限  │
│         │     │  Refresh    │     │             │     │             │
└─────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

**业务规则：**
- 密码采用 BCrypt 加密存储，强度要求：8位以上，含大小写+数字+特殊字符
- 登录失败 5 次锁定账号 30 分钟
- Access Token 有效期 2 小时，Refresh Token 有效期 7 天
- 支持 Token 黑名单（主动登出场景）

#### 2.2.2 资源管理流程（以云主机为例）

```
┌─────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  用户    │────▶│  创建云主机  │────▶│  参数校验   │────▶│  配额检查   │
│         │     │  请求       │     │  名称/规格   │     │  资源限制   │
└─────────┘     └─────────────┘     └─────────────┘     └──────┬──────┘
                                                                │
                                                                ▼
┌─────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  用户    │◀────│  返回结果   │◀────│  更新状态   │◀────│  调用 Nova  │
│         │     │  成功/失败  │     │  数据库记录  │     │  API 创建   │
└─────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

**业务规则：**
- 创建前校验：名称唯一性、规格合法性、镜像可用性、网络配置
- 配额检查：vCPU、内存、磁盘容量是否超出租户限制
- 异步处理：云主机创建为异步操作，通过 WebSocket 推送状态变更
- 状态流转：创建中 → 运行中/错误

```
│  触发告警       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  AlertManager   │────▶│  Webhook 接收   │────▶│  静默规则检查   │
│  发送告警       │     │  /api/monitor   │     │  匹配标签       │
│                 │     │  /webhook       │     │                 │
└─────────────────┘     └─────────────────┘     └────────┬────────┘
                                                         │
                              ┌──────────────────────────┼──────────────────────────┐
                              │                          │                          │
                              ▼                          ▼                          ▼
                    ┌─────────────────┐        ┌─────────────────┐        ┌─────────────────┐
                    │   命中静默规则   │        │  写入告警记录   │        │  WebSocket      │
                    │   丢弃告警      │        │  monitor_alert  │        │  实时推送       │
                    │                 │        │  _record        │        │                 │
                    └─────────────────┘        └────────┬────────┘        └─────────────────┘
                                                        │
                                                        ▼
                                               ┌─────────────────┐
                                               │  异步通知发送   │
                                               │  邮件/钉钉/企微  │
                                               │  指数退避重试   │
                                               └─────────────────┘
```

**业务规则：**
- 告警去重：基于 (alertName, instance, firedAt) 唯一索引幂等写入
- 静默匹配：多标签 AND 条件匹配，支持时间范围
- 通知策略：P0 立即通知，P1 5分钟内，P2 汇总通知
- 重试机制：最多重试 3 次，指数退避（1s → 2s → 4s）

## 四、业务规则汇总

### 4.1 通用业务规则

| **审计字段** | 所有表包含 `created_at`, `updated_at`, `created_by`, `updated_by` |
| **数据权限** | 支持全部数据、本部门数据、本人数据三级数据权限 |

### 4.2 安全相关规则

| 规则类别 | 规则内容 |
|---------|---------|
| **密码策略** | 8位以上，必须包含大小写字母、数字、特殊字符 |
| **登录锁定** | 连续 5 次失败锁定 30 分钟 |
| **会话管理** | Access Token 2 小时，Refresh Token 7 天 |
| **敏感数据** | 密钥、Token 等使用 AES-256-GCM 加密存储 |

## 五、异常场景处理

### 5.1 认证授权异常

| 异常场景 | 处理策略 |
|---------|---------|
| Token 过期 | 返回 401，前端使用 Refresh Token 刷新 |
| Token 无效 | 返回 401，跳转登录页 |
| 权限不足 | 返回 403，提示无权访问 |
| 账号锁定 | 返回 423，提示剩余锁定时间 |

### 5.2 云平台对接异常

| 异常场景 | 处理策略 |
|---------|---------|
| 云平台连接失败 | 标记集群状态为"离线"，缓存上次成功数据 |
| API 调用超时 | 重试 3 次，超时时间 30s |
| 配额不足 | 前置校验，返回明确错误信息 |
| 云资源不存在 | 同步清理本地数据库记录 |

### 5.3 监控告警异常

| 异常场景 | 处理策略 |
|---------|---------|
| Prometheus 不可达 | 返回缓存数据，标记数据源状态 |
| AlertManager 重复推送 | 幂等写入，基于唯一索引去重 |
| 通知发送失败 | 指数退避重试 3 次，记录失败日志 |
| WebSocket 断开 | 自动重连，恢复订阅状态 |
