## Token 设计

| Token 类型 | 有效期 | 存储 | 说明 |
|-----------|--------|------|------|
| Access Token | 2 小时 | 客户端 Header | 每次请求携带，含 tokenVersion |
| Refresh Token | 7 天 | 客户端 + Redis | 用于换新 Access Token |
| 黑名单 | Token 剩余有效期 | Redis `auth:blacklist:{jti}` | 登出/修改密码时写入 |

**JWT Payload 关键字段：**`sub`（userId）、`jti`（唯一 ID）、`tokenVersion`（版本号，用于强制下线）

**Token 刷新传参说明：** 刷新时需通过 Header 携带 `X-User-Id`（userId）和 `X-Old-Jti`（旧 Token jti），配合 RequestBody 的 `refreshToken` 三要素联合校验。

## 权限模型（RBAC）

```
用户 (auth_user_credential.user_id)
  └── 多角色 (sys_role) [via sys_user_role]
        └── 多菜单/权限 (sys_menu) [via sys_role_menu]
              └── permission 字段（如 system:user:list）
```

> auth 模块通过 `PermissionMapper` 直接查 system 模块共享 Schema（单体架构，无 Feign）。

`@PreAuthorize("hasAuthority('system:user:list')")` 在 Controller 方法上使用。

## 异常码规范

| 分类 | 异常码 | 含义 |
|------|--------|------|
| 验证码 | AUTH_1001 | 验证码不存在或已过期 |
| 验证码 | AUTH_1002 | 验证码错误 |
| 登录 | AUTH_2001 | 用户名或密码错误 |
| 登录 | AUTH_2002 | 账号已被禁用 |
| 登录 | AUTH_2003 | 账号已被锁定，请 X 分钟后重试 |
| 登录 | AUTH_2004 | 登录请求过于频繁，请稍后再试 |
| 密码 | AUTH_3001 | 旧密码不正确 |
| 密码 | AUTH_3002 | 密码强度不足 |
| 密码 | AUTH_3003 | 新密码与旧密码相同 |
| 密码 | AUTH_3004 | 两次输入密码不一致 |
| 账号 | AUTH_4001 | 用户不存在 |
| 账号 | AUTH_4002 | 账号当前未处于锁定状态 |
| Token | AUTH_5001 | Token 已失效或被吊销 |
| Token | AUTH_5002 | Token 已过期 |
