# opencloud-auth 数据设计文档

---

## 1. 数据库选型

| 存储 | 用途 |
|------|------|
| PostgreSQL | 用户凭证、登录日志、操作日志等持久化数据 |
| Redis | Token 黑名单、RefreshToken、Token 版本号、权限缓存、验证码、登录限频计数器 |

Schema 名：`auth`

---

## 2. 数据表设计

### 2.1 用户凭证表 `auth_user_credential`

> 存储用户的认证凭证，与 system 模块的 `sys_user` 通过 `user_id` 关联但解耦。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | bigserial | PK | 自增主键 |
| user_id | varchar(36) | NOT NULL, UNIQUE | 关联 sys_user.id |
| tenant_id | varchar(36) | NOT NULL | 租户 ID |
| username | varchar(64) | NOT NULL | 登录账号（租户内唯一） |
| password_hash | varchar(128) | NOT NULL | BCrypt 加密密码（强度 12） |
| status | varchar(16) | NOT NULL, DEFAULT 'ACTIVE' | 账号状态：ACTIVE / DISABLED / LOCKED |
| login_fail_count | smallint | NOT NULL, DEFAULT 0 | 连续登录失败次数（≥5 次触发锁定） |
| lock_expire_time | timestamptz | NULL | 锁定解除时间；状态自动解锁时依据此字段 |
| last_login_time | timestamptz | NULL | 最近登录成功时间（登录成功后更新） |
| password_update_time | timestamptz | NULL | 最近修改密码时间 |
| created_at | timestamptz | NOT NULL, DEFAULT now() | 创建时间 |
| updated_at | timestamptz | NOT NULL, DEFAULT now() | 更新时间 |
| deleted_at | timestamptz | NULL | 软删除时间（逻辑删除） |

**字段说明：**
- `status = LOCKED` + `lock_expire_time <= now()` → 下次登录时自动解锁（无需定时任务）
- `login_fail_count` 在登录成功后清零，在密码比对失败时 +1；管理员解锁时也清零

**索引：**
```sql
CREATE UNIQUE INDEX idx_auc_username ON auth_user_credential(username) WHERE deleted_at IS NULL;
CREATE INDEX idx_auc_tenant ON auth_user_credential(tenant_id);
CREATE INDEX idx_auc_user_id ON auth_user_credential(user_id);
```

### 2.2 登录日志表 `auth_login_log`

> 记录每次登录 / 登出行为，用于审计和异常分析。只追加，不修改（login_time / logout_time 在同一行，登出时补填）。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | bigserial | PK | 自增主键 |
| user_id | varchar(36) | NOT NULL | 用户 ID |
| tenant_id | varchar(36) | NOT NULL | 租户 ID |
| username | varchar(64) | NOT NULL | 用户名（冗余，方便日志查询） |
| login_ip | varchar(64) | NULL | 客户端 IP（取 X-Forwarded-For 或 RemoteAddr） |
| user_agent | varchar(512) | NULL | 浏览器 / 客户端标识 |
| login_time | timestamptz | NULL | 登录时间（登录事件写入） |
| logout_time | timestamptz | NULL | 登出时间（登出 / 强制下线时补填） |
| status | varchar(16) | NOT NULL | 事件状态：SUCCESS / FAILED / LOGOUT / FORCE_LOGOUT |
| fail_reason | varchar(256) | NULL | 失败原因（PASSWORD_ERROR / ACCOUNT_LOCKED 等），仅 status=FAILED 时有值 |
| created_at | timestamptz | NOT NULL, DEFAULT now() | 记录创建时间 |

**索引：**
```sql
CREATE INDEX idx_all_user ON auth_login_log(user_id, created_at DESC);
CREATE INDEX idx_all_tenant ON auth_login_log(tenant_id, created_at DESC);
CREATE INDEX idx_all_status ON auth_login_log(status);
```

**数据保留策略：** 保留最近 90 天数据，超期通过定时任务归档或删除。

---

### 2.3 操作日志表 `auth_operation_log`

> 记录修改密码、强制下线、解锁账号等关键管理操作，用于安全审计。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | bigserial | PK | 自增主键 |
| operator_id | varchar(36) | NOT NULL | 操作人用户 ID（管理员或本人） |
| tenant_id | varchar(36) | NOT NULL | 操作人所属租户 ID |
| target_user_id | varchar(36) | NULL | 被操作用户 ID（自操作时与 operator_id 相同） |
| operation_type | varchar(32) | NOT NULL | 操作类型：CHANGE_PASSWORD / FORCE_LOGOUT / UNLOCK_ACCOUNT |
| operate_time | timestamptz | NOT NULL, DEFAULT now() | 操作时间 |
| operate_ip | varchar(64) | NULL | 操作人客户端 IP |
| remark | varchar(256) | NULL | 附加说明（可选） |

**operation_type 枚举值：**

| 值 | 触发场景 |
|----|---------|
| `CHANGE_PASSWORD` | 用户修改自己的密码（POST /auth/password/change） |
| `FORCE_LOGOUT` | 管理员强制下线指定用户（POST /auth/admin/force-logout/{userId}） |
| `UNLOCK_ACCOUNT` | 管理员手动解锁锁定账号（POST /auth/admin/unlock/{userId}） |

**索引：**
```sql
CREATE INDEX idx_aol_operator ON auth_operation_log(operator_id, operate_time DESC);
CREATE INDEX idx_aol_target ON auth_operation_log(target_user_id, operate_time DESC);
CREATE INDEX idx_aol_type ON auth_operation_log(operation_type);
```

**数据保留策略：** 操作日志永久保留（或按合规要求归档）。

### 2.4 扩展表（预留，当前版本不启用）

以下两张表为后续扩展预留，当前业务范围（账号密码登录）不涉及，不在 v1 版本初始化：

#### `auth_oauth_client` — OAuth2 客户端表
> 用于 API 调用方或第三方系统接入，待 OAuth2 授权流程纳入业务范围后启用。

#### `auth_social_bind` — 第三方账号绑定表
> 用于飞书、企微、GitHub 等第三方 OAuth2 登录，待社交登录需求确认后启用。

## 3. Redis 数据结构说明

> 所有 Key 以 `auth:` 为统一命名空间前缀，与业务设计和应用设计保持一致。

| Key 模式 | 类型 | 示例值 | TTL | 说明 |
|----------|------|--------|-----|------|
| `auth:captcha:{captchaKey}` | String | `"8f3k"` | 5 分钟 | 图形验证码文本（统一小写），一次有效，校验后立即删除 |
| `auth:refresh:{userId}:{jti}` | String | RefreshToken UUID 值 | 7 天 | RefreshToken 存储，jti 与 AccessToken jti 对应；刷新时旧 Key 立即删除 |
| `auth:blacklist:{jti}` | String | `"1"` | AccessToken 剩余有效期 | Token 黑名单；登出 / 修改密码 / 强制下线时写入 |
| `auth:perm:{userId}` | Set / String | 权限标识集合 | **10 分钟** | 用户权限集合缓存；角色变更时由 system 模块主动清除 |
| `auth:token:version:{userId}` | String | `"3"` | 永久（账号注销时清除） | Token 版本号；强制下线时 incr +1，所有旧 AccessToken 自动失效 |
| `auth:login:rate:{ip}` | String | `"7"` | 1 分钟 | 同一 IP 登录请求计数器；≥10 次返回 429；窗口期结束自动过期 |

**Redis Key 操作规范：**

| 场景 | Redis 操作 |
|------|-----------|
| 登录成功 | `SET auth:refresh:{userId}:{jti}`（TTL 7d）；读取 `auth:token:version:{userId}` |
| 登出 | `SET auth:blacklist:{jti}`（TTL=剩余有效期）；`DEL auth:refresh:{userId}:{jti}` |
| Token 刷新 | `DEL auth:refresh:{userId}:{oldJti}`；`SET auth:refresh:{userId}:{newJti}` |
| 修改密码 | `SET auth:blacklist:{currentJti}`；`DEL auth:refresh:{userId}:*`（SCAN 删除） |
| 强制下线 | `INCR auth:token:version:{userId}`；`DEL auth:refresh:{userId}:*` |
| 管理员解锁 | 不涉及 Redis 操作（仅更新 DB） |
| 权限缓存失效 | `DEL auth:perm:{userId}`（由 system 模块调用） |

## 4. 数据流向说明

```
用户登录（POST /auth/login）
  ↓
读 auth_user_credential（校验状态、密码比对、fail_count 更新）
  ↓
读 Redis auth:token:version:{userId}（获取 tokenVersion）
  ↓
生成 AccessToken + RefreshToken
  ↓
写 Redis auth:refresh:{userId}:{jti}（存储 RefreshToken）
  ↓
写 auth_login_log（status=SUCCESS / FAILED）

──────────────────────────────────────────

用户登出（POST /auth/logout）
  ↓
写 Redis auth:blacklist:{jti}（Token 黑名单）
  ↓
删 Redis auth:refresh:{userId}:{jti}
  ↓
写 auth_login_log（status=LOGOUT，补填 logout_time）

──────────────────────────────────────────

Token 刷新（POST /auth/token/refresh）
  ↓
校验 + 删 Redis auth:refresh:{userId}:{oldJti}
  ↓
读 Redis auth:token:version:{userId}
  ↓
生成新 AccessToken + RefreshToken
  ↓
写 Redis auth:refresh:{userId}:{newJti}

──────────────────────────────────────────

修改密码（POST /auth/password/change）
  ↓
读 auth_user_credential（BCrypt 比对旧密码）
  ↓
更新 auth_user_credential.password_hash / password_update_time
  ↓
删 Redis auth:refresh:{userId}:*（清除所有 RefreshToken）
  ↓
写 Redis auth:blacklist:{currentJti}（当前 Token 失效）
  ↓
写 auth_operation_log（operation_type=CHANGE_PASSWORD）

──────────────────────────────────────────

强制下线（POST /auth/admin/force-logout/{userId}）
  ↓
INCR Redis auth:token:version:{userId}（所有旧 AccessToken 失效）
  ↓
删 Redis auth:refresh:{userId}:*
  ↓
写 auth_login_log（status=FORCE_LOGOUT，补填 logout_time）
  ↓
写 auth_operation_log（operation_type=FORCE_LOGOUT）

──────────────────────────────────────────

解锁账号（POST /auth/admin/unlock/{userId}）
  ↓
更新 auth_user_credential（status=ACTIVE，fail_count=0，lock_expire_time=null）
  ↓
写 auth_operation_log（operation_type=UNLOCK_ACCOUNT）

──────────────────────────────────────────

权限鉴权（JwtAuthenticationFilter）
  ↓
读 Redis auth:blacklist:{jti}（黑名单检查）
  ↓
读 Redis auth:token:version:{userId}（版本比对）
  ↓
读 Redis auth:perm:{userId}（权限缓存命中直接返回）
  ↓
（缓存未命中）→ 查 opencloud-system 角色权限 → 写 Redis auth:perm:{userId}（TTL 10min）
```

## 5. 数据初始化脚本（DDL）

```sql
-- ============================================================
-- Schema
-- ============================================================
CREATE SCHEMA IF NOT EXISTS auth;

-- ============================================================
-- 用户凭证表
-- ============================================================
CREATE TABLE auth.auth_user_credential (
    id                   bigserial    PRIMARY KEY,
    user_id              varchar(36)  NOT NULL,
    tenant_id            varchar(36)  NOT NULL,
    username             varchar(64)  NOT NULL,
    password_hash        varchar(128) NOT NULL,
    status               varchar(16)  NOT NULL DEFAULT 'ACTIVE',  -- ACTIVE / DISABLED / LOCKED
    login_fail_count     smallint     NOT NULL DEFAULT 0,
    lock_expire_time     timestamptz,
    last_login_time      timestamptz,
    password_update_time timestamptz,
    created_at           timestamptz  NOT NULL DEFAULT now(),
    updated_at           timestamptz  NOT NULL DEFAULT now(),
    deleted_at           timestamptz
);

CREATE UNIQUE INDEX idx_auc_username  ON auth.auth_user_credential(username)  WHERE deleted_at IS NULL;
CREATE UNIQUE INDEX idx_auc_user_id   ON auth.auth_user_credential(user_id)   WHERE deleted_at IS NULL;
CREATE INDEX        idx_auc_tenant    ON auth.auth_user_credential(tenant_id);

COMMENT ON TABLE  auth.auth_user_credential                      IS '用户认证凭证表';
COMMENT ON COLUMN auth.auth_user_credential.status              IS '账号状态: ACTIVE=正常 DISABLED=禁用 LOCKED=锁定';
COMMENT ON COLUMN auth.auth_user_credential.login_fail_count    IS '连续登录失败次数，≥5 次触发锁定，登录成功后清零';
COMMENT ON COLUMN auth.auth_user_credential.lock_expire_time    IS '锁定解除时间，到期后下次登录自动解锁';
COMMENT ON COLUMN auth.auth_user_credential.password_update_time IS '最近一次修改密码时间';

-- ============================================================
-- 登录日志表
-- ============================================================
CREATE TABLE auth.auth_login_log (
    id          bigserial    PRIMARY KEY,
    user_id     varchar(36)  NOT NULL,
    tenant_id   varchar(36)  NOT NULL,
    username    varchar(64)  NOT NULL,
    login_ip    varchar(64),
    user_agent  varchar(512),
    login_time  timestamptz,
    logout_time timestamptz,
    status      varchar(16)  NOT NULL,  -- SUCCESS / FAILED / LOGOUT / FORCE_LOGOUT
    fail_reason varchar(256),
    created_at  timestamptz  NOT NULL DEFAULT now()
);

CREATE INDEX idx_all_user   ON auth.auth_login_log(user_id,   created_at DESC);
CREATE INDEX idx_all_tenant ON auth.auth_login_log(tenant_id, created_at DESC);
CREATE INDEX idx_all_status ON auth.auth_login_log(status);

COMMENT ON TABLE  auth.auth_login_log           IS '登录/登出日志表，只追加写入';
COMMENT ON COLUMN auth.auth_login_log.status    IS '事件状态: SUCCESS / FAILED / LOGOUT / FORCE_LOGOUT';
COMMENT ON COLUMN auth.auth_login_log.logout_time IS '登出时间，登出/强制下线时补填';

-- ============================================================
-- 操作日志表
-- ============================================================
CREATE TABLE auth.auth_operation_log (
    id              bigserial    PRIMARY KEY,
    operator_id     varchar(36)  NOT NULL,
    tenant_id       varchar(36)  NOT NULL,
    target_user_id  varchar(36),
    operation_type  varchar(32)  NOT NULL,  -- CHANGE_PASSWORD / FORCE_LOGOUT / UNLOCK_ACCOUNT
    operate_time    timestamptz  NOT NULL DEFAULT now(),
    operate_ip      varchar(64),
    remark          varchar(256)
);

CREATE INDEX idx_aol_operator ON auth.auth_operation_log(operator_id,    operate_time DESC);
CREATE INDEX idx_aol_target   ON auth.auth_operation_log(target_user_id, operate_time DESC);
CREATE INDEX idx_aol_type     ON auth.auth_operation_log(operation_type);

COMMENT ON TABLE  auth.auth_operation_log                IS '关键管理操作审计日志表';
COMMENT ON COLUMN auth.auth_operation_log.operation_type IS 'CHANGE_PASSWORD=修改密码 FORCE_LOGOUT=强制下线 UNLOCK_ACCOUNT=解锁账号';
```

## 6. 表与业务流程对照

| 业务流程 | 读表 | 写表 | Redis 操作 |
|---------|------|------|-----------|
| 用户登录 | `auth_user_credential` | `auth_user_credential`（更新 fail_count / last_login_time）、`auth_login_log` | SET refresh、读 token:version |
| Token 刷新 | — | — | DEL 旧 refresh、SET 新 refresh |
| 用户登出 | — | `auth_login_log`（补 logout_time） | SET blacklist、DEL refresh |
| 获取当前用户 | `auth_user_credential`（via system 模块） | — | 读/写 perm 缓存 |
| 修改密码 | `auth_user_credential` | `auth_user_credential`（更新 password_hash）、`auth_operation_log` | SET blacklist、DEL refresh:* |
| 强制下线 | — | `auth_login_log`（补 logout_time）、`auth_operation_log` | INCR token:version、DEL refresh:* |
| 解锁账号 | `auth_user_credential` | `auth_user_credential`（更新 status/fail_count）、`auth_operation_log` | — |
| 权限鉴权 | — | — | 读 blacklist、读 token:version、读/写 perm |
| 验证码 | — | — | SET captcha / GET+DEL captcha |
| 登录限频 | — | — | INCR login:rate:{ip} |
