# opencloud-system 应用设计文档

---

## 1. 模块定位

`opencloud-system` 是平台系统管理模块，职责边界：

- **对外**：提供用户管理、角色管理、菜单权限管理、部门管理、字典管理、租户管理、个人中心、操作日志、登录日志查询等 REST 接口
- **对内**：维护 RBAC 核心数据（sys_user / sys_role / sys_menu / sys_user_role / sys_role_menu），以及操作日志（sys_operation_log）
- **依赖**：PostgreSQL（持久化数据）、Redis（清除 auth 权限缓存）、opencloud-auth（依赖 auth 模块的安全过滤器与 SecurityContext）
- **被依赖**：opencloud-auth 查询 sys_user / sys_role_menu 等表获取用户权限

## 5. 关键设计说明

### 5.1 SysLog 注解与 AOP 切面

```java
// 注解定义
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface SysLog {
    String module() default "";   // 模块名，如"用户管理"
    String action() default "";   // 操作描述，如"新增用户"
}

// 使用示例
@PostMapping
@SysLog(module = "用户管理", action = "新增用户")
public Result<UserVO> createUser(@RequestBody UserCreateDTO dto) { ... }
```

**AOP 切面处理逻辑：**
1. `@Before`：记录请求开始时间；从 `HttpServletRequest` 获取 IP、URL、Method、请求体（脱敏）
2. `@AfterReturning`：记录响应结果（success），计算耗时
3. `@AfterThrowing`：记录异常信息（exceptionMsg），status=FAILED
4. 异步写入（`@Async`）：通过 `ApplicationEventPublisher` 发布日志事件，避免切面线程阻塞

### 5.2 AuthPermCacheHelper（权限缓存清除）

封装所有清除 auth Redis 权限缓存的操作，统一管理：

```java
@Component
public class AuthPermCacheHelper {

    private static final String PERM_KEY_PREFIX = "auth:perm:";
    private static final String TOKEN_VERSION_KEY = "auth:token:version:";
    private static final String REFRESH_KEY_PREFIX = "auth:refresh:";

    // 清除单个用户权限缓存
    public void evictPerm(String userId) {
        redisTemplate.delete(PERM_KEY_PREFIX + userId);
    }

    // 批量清除多个用户权限缓存
    public void evictPerms(List<String> userIds) {
        List<String> keys = userIds.stream()
            .map(id -> PERM_KEY_PREFIX + id)
            .collect(Collectors.toList());
        redisTemplate.delete(keys);
    }

    // 踢出用户（强制所有 Token 失效 + 清除 RefreshToken）
    public void forceLogout(String userId) {
        redisTemplate.opsForValue().increment(TOKEN_VERSION_KEY + userId);
        // SCAN 删除所有 RefreshToken
        ScanOptions options = ScanOptions.scanOptions()
            .match(REFRESH_KEY_PREFIX + userId + ":*").count(100).build();
        // ... scan 删除逻辑
    }
}
```

### 5.4 租户隔离拦截

通过 MyBatis-Plus 拦截器（InnerInterceptor）在 SELECT / UPDATE / DELETE 中自动注入 `tenant_id = ?` 条件，避免每个 Mapper 手动写租户过滤：

```java
// 自定义租户处理器
@Component
public class SystemTenantHandler implements TenantLineHandler {
    @Override
    public Expression getTenantId() {
        // 从 SecurityContext 获取当前用户的 tenantId
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        return new StringValue(auth.getDetails().getTenantId());
    }

    @Override
    public boolean ignoreTable(String tableName) {
        // 以下表不做租户过滤（超级管理员全局查看）
        return Set.of("sys_tenant", "sys_menu", "sys_dict_type", "sys_dict_data")
                  .contains(tableName);
    }
}
```

> **说明：** `sys_menu`、`sys_dict_*` 为平台全局共享数据，不做租户过滤；`sys_user`、`sys_role`、`sys_dept` 等需要租户隔离。

## 6. 安全设计

### 6.1 接口权限控制

- 所有 `/api/system/**` 接口均需 JWT 认证（由 auth 模块的 `JwtAuthenticationFilter` 统一处理）
- 接口级别权限通过 `@PreAuthorize("hasAuthority('system:user:add')")` 控制
- 超级管理员（SUPER_ADMIN 角色）跳过权限校验，直接放行
- 个人中心接口（`/api/system/profile/**`）仅需登录，不需要特定权限

### 6.2 租户隔离

- MyBatis-Plus TenantLineInnerInterceptor 自动注入 tenant_id 过滤（见 5.4）
- 不在拦截范围内的手动查询，Service 层显式附加 tenantId 参数
- 超级管理员操作租户管理时，临时旁路租户过滤（由 TenantHandler.ignoreTable 控制）

### 6.3 操作限制

- admin 账号（is_builtin=true）受保护，不允许删除/禁用（Service 层校验）
- 管理员不允许对自身账号执行禁用/删除
- 内置角色（is_builtin=true）不允许删除，不允许修改 role_code

## 7. 异常处理

### 7.1 异常码规范

| 错误码范围 | 模块 |
|-----------|------|
| SYS_1XXX | 用户管理 |
| SYS_2XXX | 角色管理 |
| SYS_3XXX | 菜单管理 |
| SYS_4XXX | 部门管理 |
| SYS_5XXX | 字典管理 |
| SYS_6XXX | 租户管理 |
| SYS_7XXX | 个人中心 |

### 7.2 SystemException 定义

```java
@Getter
public class SystemException extends RuntimeException {
    private final String code;

    public SystemException(String code, String message) {
        super(message);
        this.code = code;
    }

    // 工厂方法
    public static SystemException userNotFound() {
        return new SystemException("SYS_1002", "用户不存在");
    }

    public static SystemException roleInUse() {
        return new SystemException("SYS_2004", "角色下存在用户，请先解绑");
    }
    // ... 其他工厂方法
}
```

### 7.3 统一异常捕获

复用 opencloud-common 的 `GlobalExceptionHandler`，`SystemException` 按照 `Result.fail(code, message)` 格式响应。
