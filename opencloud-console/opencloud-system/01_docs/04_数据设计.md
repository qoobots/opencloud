# opencloud-system 数据设计文档

---

## 1. 数据库选型

| 存储 | 用途 |
|------|------|
| PostgreSQL | 用户、角色、菜单、部门、租户、字典、操作日志等所有持久化数据 |
| Redis | 仅用于清除 auth 模块的权限缓存（`auth:perm:{userId}`），不维护 system 自身缓存 |

Schema 名：`system`

---

## 2. 数据表设计

### 2.1 用户表 `sys_user`

> 存储用户基础信息（不含认证凭证，认证凭证由 auth 模块的 `auth_user_credential` 维护）。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | varchar(36) | PK | 用户 ID（UUID） |
| tenant_id | varchar(36) | NOT NULL | 所属租户 ID |
| dept_id | bigint | NULL | 所属部门 ID，关联 sys_dept.id（可为 null） |
| username | varchar(64) | NOT NULL | 登录账号，同租户内唯一 |
| nickname | varchar(64) | NOT NULL | 显示昵称 |
| avatar | varchar(512) | NULL | 头像 URL |
| email | varchar(128) | NULL | 邮箱（选填） |
| phone | varchar(16) | NULL | 手机号（选填） |
| status | varchar(16) | NOT NULL, DEFAULT 'ACTIVE' | 状态：ACTIVE / DISABLED |
| is_builtin | boolean | NOT NULL, DEFAULT false | 是否内置账号（admin=true，不可删除禁用） |
| sort | integer | NOT NULL, DEFAULT 0 | 排序 |
| created_at | timestamptz | NOT NULL, DEFAULT now() | 创建时间 |
| updated_at | timestamptz | NOT NULL, DEFAULT now() | 更新时间 |
| deleted | smallint | NOT NULL, DEFAULT 0 | 逻辑删除（0=未删，1=已删） |

**索引：**
```sql
CREATE UNIQUE INDEX idx_su_username_tenant ON sys_user(username, tenant_id) WHERE deleted = 0;
CREATE INDEX idx_su_tenant ON sys_user(tenant_id);
CREATE INDEX idx_su_dept ON sys_user(dept_id);
CREATE INDEX idx_su_status ON sys_user(status);
```

### 2.4 菜单/权限表 `sys_menu`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | bigserial | PK | 菜单 ID |
| parent_id | bigint | NOT NULL, DEFAULT 0 | 父节点 ID（0=顶层） |
| menu_name | varchar(64) | NOT NULL | 菜单/按钮名称 |
| menu_type | smallint | NOT NULL | 类型：0=目录 1=菜单 2=按钮 |
| path | varchar(256) | NULL | 前端路由 path（目录/菜单有值） |
| component | varchar(256) | NULL | 前端组件路径（菜单有值） |
| icon | varchar(64) | NULL | 菜单图标 |
| permission | varchar(128) | NULL | 权限标识（按钮节点必填，如 system:user:add） |
| visible | boolean | NOT NULL, DEFAULT true | 是否在侧边栏显示 |
| sort | integer | NOT NULL, DEFAULT 0 | 同级排序 |
| remark | varchar(256) | NULL | 备注 |
| created_at | timestamptz | NOT NULL, DEFAULT now() | 创建时间 |
| updated_at | timestamptz | NOT NULL, DEFAULT now() | 更新时间 |
| deleted | smallint | NOT NULL, DEFAULT 0 | 逻辑删除 |

**约束：**
- `menu_type=2`（按钮）时，`path` 和 `component` 应为 NULL
- `permission` 全平台唯一（应用层校验）

**索引：**
```sql
CREATE INDEX idx_sm_parent ON sys_menu(parent_id);
CREATE INDEX idx_sm_type ON sys_menu(menu_type);
CREATE UNIQUE INDEX idx_sm_permission ON sys_menu(permission) WHERE deleted = 0 AND permission IS NOT NULL;
```

### 2.10 操作日志表 `sys_operation_log`

> 由 `@SysLog` AOP 切面异步写入，只追加不修改。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | bigserial | PK | 日志 ID |
| user_id | varchar(36) | NULL | 操作人用户 ID（未登录时为 null） |
| tenant_id | varchar(36) | NULL | 操作人所属租户 ID |
| username | varchar(64) | NULL | 操作人用户名（冗余，日志快速查询） |
| module_name | varchar(64) | NOT NULL | 功能模块名（如"用户管理"） |
| operation_type | varchar(64) | NOT NULL | 操作描述（如"新增用户"） |
| method | varchar(8) | NOT NULL | HTTP 方法（GET/POST/PUT/DELETE） |
| request_url | varchar(512) | NOT NULL | 请求 URL |
| request_param | text | NULL | 请求参数（JSON 格式，已脱敏） |
| response_result | text | NULL | 响应结果摘要（超长截断） |
| exception_msg | text | NULL | 异常信息（仅 status=FAILED 时有值） |
| status | varchar(16) | NOT NULL | 执行结果：SUCCESS / FAILED |
| cost_time | integer | NULL | 接口耗时（毫秒） |
| operate_ip | varchar(64) | NULL | 操作人 IP |
| operate_time | timestamptz | NOT NULL, DEFAULT now() | 操作时间 |

**索引：**
```sql
CREATE INDEX idx_sol_user ON sys_operation_log(user_id, operate_time DESC);
CREATE INDEX idx_sol_tenant ON sys_operation_log(tenant_id, operate_time DESC);
CREATE INDEX idx_sol_module ON sys_operation_log(module_name);
CREATE INDEX idx_sol_status ON sys_operation_log(status);
CREATE INDEX idx_sol_time ON sys_operation_log(operate_time DESC);
```

**数据保留策略：** 保留最近 180 天，定时任务（每日凌晨 2:00）删除过期记录。

## 3. 跨 Schema 访问

`opencloud-system` 需要访问 `auth` Schema 中的 `auth_login_log` 表（只读），使用全限定表名：

```sql
-- 登录日志分页查询（跨 Schema）
SELECT * FROM auth.auth_login_log
WHERE tenant_id = ?
  AND username LIKE ?
  AND status = ?
  AND login_time BETWEEN ? AND ?
ORDER BY login_time DESC
LIMIT ? OFFSET ?;
```

在 MyBatis-Plus 的 Mapper XML 中使用 `auth.auth_login_log` 的全限定表名即可（不需要额外 datasource 配置）。

## 4. 数据流向说明

```
创建用户
  ↓
写 system.sys_user（基础信息）
  ↓
写 auth.auth_user_credential（认证凭证，BCrypt 密码）
  ↓
写 system.sys_user_role（角色绑定，若指定）
  ↓
（无需清除缓存，新用户首次登录时缓存自然生成）

──────────────────────────────────────────

分配用户角色
  ↓
删除 system.sys_user_role（该用户原有绑定）
  ↓
批量写 system.sys_user_role（新角色绑定）
  ↓
DEL Redis auth:perm:{userId}（清除权限缓存）

──────────────────────────────────────────

分配角色菜单
  ↓
删除 system.sys_role_menu（该角色原有绑定）
  ↓
批量写 system.sys_role_menu（新菜单绑定）
  ↓
查询该角色下所有用户 ID（system.sys_user_role WHERE role_id = ?）
  ↓
批量 DEL Redis auth:perm:{userId}

──────────────────────────────────────────

禁用用户
  ↓
更新 system.sys_user.status = DISABLED
  ↓
更新 auth.auth_user_credential.status = DISABLED
  ↓
DEL Redis auth:perm:{userId}
  ↓
INCR Redis auth:token:version:{userId}
  ↓
SCAN+DEL Redis auth:refresh:{userId}:*

──────────────────────────────────────────

重置用户密码
  ↓
更新 auth.auth_user_credential.password_hash / password_update_time
  ↓
INCR Redis auth:token:version:{userId}
  ↓
SCAN+DEL Redis auth:refresh:{userId}:*

──────────────────────────────────────────

禁用租户
  ↓
更新 system.sys_tenant.status = DISABLED
  ↓
查询租户下所有用户 ID（system.sys_user WHERE tenant_id = ?）
  ↓
批量 DEL Redis auth:perm:{userId}
  ↓
批量 INCR Redis auth:token:version:{userId}

──────────────────────────────────────────

操作日志记录
  ↓
@SysLog AOP 切面拦截（@Async）
  ↓
写 system.sys_operation_log
```

## 6. 表与业务流程对照

| 业务流程 | 读表 | 写表 | Redis 操作 |
|---------|------|------|-----------|
| 用户分页查询 | `sys_user`、`sys_dept` | — | — |
| 新增用户 | `sys_user`（唯一性检查） | `sys_user`、`auth_user_credential`、`sys_user_role` | — |
| 编辑用户 | `sys_user` | `sys_user` | — |
| 删除用户 | `sys_user` | `sys_user`（逻辑删）、`auth_user_credential`（逻辑删）、`sys_user_role`（物理删） | DEL auth:perm:{userId} |
| 启用/禁用用户 | `sys_user` | `sys_user`、`auth_user_credential` | 禁用时 DEL perm + INCR version + SCAN DEL refresh |
| 重置密码 | — | `auth_user_credential` | INCR version + SCAN DEL refresh |
| 分配角色 | `sys_role` | `sys_user_role` | DEL auth:perm:{userId} |
| 角色分页查询 | `sys_role` | — | — |
| 新增角色 | `sys_role`（唯一性检查） | `sys_role` | — |
| 删除角色 | `sys_user_role`（检查绑定） | `sys_role`（逻辑删）、`sys_role_menu`（物理删） | — |
| 分配菜单权限 | `sys_user_role`（获取受影响用户） | `sys_role_menu` | 批量 DEL auth:perm:{userId} |
| 菜单树查询 | `sys_menu` | — | — |
| 当前用户菜单树 | `sys_user_role`、`sys_role_menu`、`sys_menu` | — | — |
| 新增/编辑菜单 | `sys_menu` | `sys_menu` | 编辑 permission 时批量 DEL auth:perm |
| 删除菜单 | `sys_role_menu`（检查绑定）、`sys_menu`（检查子节点） | `sys_menu`（逻辑删）、`sys_role_menu` | 批量 DEL auth:perm |
| 部门树查询 | `sys_dept` | — | — |
| 新增/编辑/删除部门 | `sys_dept`、`sys_user`（检查归属） | `sys_dept` | — |
| 字典查询 | `sys_dict_type`、`sys_dict_data` | — | — |
| 字典维护 | `sys_dict_type`、`sys_dict_data` | `sys_dict_type`、`sys_dict_data` | — |
| 新增租户 | `sys_tenant`（唯一性） | `sys_tenant`、`sys_user`、`auth_user_credential` | — |
| 禁用租户 | `sys_user`（查用户列表） | `sys_tenant` | 批量 DEL perm + INCR version |
| 个人信息查询 | `sys_user`、`sys_role` | — | — |
| 个人信息修改 | — | `sys_user` | — |
| 操作日志查询 | `sys_operation_log` | — | — |
| 操作日志写入（AOP） | — | `sys_operation_log`（异步） | — |
| 登录日志查询 | `auth.auth_login_log` | — | — |
