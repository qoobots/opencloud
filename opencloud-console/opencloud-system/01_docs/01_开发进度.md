## 技术关键点

| 关键点 | 实现方式 |
|--------|----------|
| 多租户隔离 | MyBatis-Plus `TenantLineInnerInterceptor` + `TenantContext`（ThreadLocal） |
| 租户上下文传递 | `TenantContextFilter` 从 `request.getAttribute("jwtClaims")` 提取 tenantId |
| 当前用户 ID 获取 | `SecurityContext.getPrincipal() instanceof String userId`（principal 为 String） |
| 权限缓存清除 | `AuthPermCacheHelper` 操作 Redis，Key = `auth:perm:{userId}` |
| 跨 Schema 写操作 | `UserCredentialUpdater`（JdbcTemplate）操作 `auth.auth_user_credential` |
| 跨 Schema 读操作 | `@TableName("auth.auth_login_log")` + 只读 Mapper |
| 操作日志记录 | `@SysLog` 注解 + `SysLogAspect` AOP，`@Async` 异步写入 |
| 树形结构构建 | `TreeBuilder<T extends TreeNode<T>>`，MenuTreeVO/DeptTreeVO 实现 TreeNode 接口 |
| 主键策略 | SysUser/SysTenant：`ASSIGN_UUID`（String），其他表：`AUTO`（Long） |
| SysUser/SysTenant 不继承 BaseEntity | BaseEntity.id 为 Long，而这两个表 id 是 varchar(36) |

## 注意事项

- **`@SysLog` 注解**命名为 `SysLog`（非原规划的 `@OperationLog`），避免与 auth 模块日志类冲突
- **SysUser / SysTenant** 不继承 `BaseEntity`（BaseEntity.id 为 Long），独立定义，字段名使用 `createdAt/updatedAt`
- **权限缓存 Key**：`auth:perm:{userId}`，TTL 10 分钟（与 auth 模块一致）
- **登录日志**：system 模块不负责写入，只做读取（通过 `@TableName("auth.auth_login_log")` 跨 Schema 查询）
- **部门层级**：通过 PostgreSQL 递归 CTE 查询层级深度，最大层级限制 5 级
- **EasyExcel 导出**：单次最多导出 10000 条，通过 `LIMIT 10000` 在 SQL 层限制
