# opencloud-web 应用设计文档

> 模块定位：主应用入口模块，聚合所有子模块，提供统一启动、全局配置、数据库管理、运维监控与容器化部署能力。
>
> 版本：v1.0 | 最后更新：2026-03-31

---

## 1. 模块定位

`opencloud-web` 是 OpenCloud 控制台的 **主应用启动与配置中枢**，职责边界：

- **对外**：暴露 `/actuator/health`（健康检查）、`/actuator/prometheus`（指标采集）、`/doc.html`（API 文档）等运维管理接口
- **对内**：提供全局 Bean 配置（WebMvc、Redis 序列化、MyBatis-Plus、异步/调度线程池、Swagger 分组）；维护 Flyway 版本迁移脚本；配置日志、CORS、全局异常处理
- **依赖**：所有子模块（opencloud-auth / opencloud-system / opencloud-cloud / opencloud-monitor / opencloud-common）


## 4. 全局配置类设计

### 4.1 WebMvcConfig — 跨域 & 拦截器

**职责：** CORS 策略配置、全局拦截器注册（请求日志 traceId 注入）

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
                .allowedOriginPatterns(allowedOrigins)   // 从配置读取
                .allowedMethods("GET","POST","PUT","DELETE","PATCH","OPTIONS")
                .allowedHeaders("Content-Type","Authorization","X-User-Id","X-Old-Jti")
                .exposedHeaders("Authorization")
                .allowCredentials(false)
                .maxAge(3600);
    }

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new TraceIdInterceptor())  // traceId 注入 MDC
                .addPathPatterns("/**")
                .excludePathPatterns("/actuator/**");
    }
}
```

**关键设计点：**
- `allowedOrigins` 从 `application.yml` 中 `opencloud.web.allowed-origins` 读取，支持多个域名逗号分隔
- 开发环境（dev profile）默认 `*`，生产环境必须配置明确域名
- CORS 配置仅覆盖 `/api/**` 路径，防止 Actuator 端点也被跨域请求

### 4.2 RedisConfig — Redis 序列化

**职责：** 配置 `RedisTemplate<String, Object>` 使用 Jackson2 JSON 序列化，替代默认的 JDK 序列化。

```java
@Configuration
public class RedisConfig {

    @Bean
    public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory factory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(factory);
        // Key: String 序列化
        template.setKeySerializer(new StringRedisSerializer());
        template.setHashKeySerializer(new StringRedisSerializer());
        // Value: Jackson2 JSON 序列化（包含类型信息）
        Jackson2JsonRedisSerializer<Object> jsonSerializer =
            new Jackson2JsonRedisSerializer<>(objectMapper(), Object.class);
        template.setValueSerializer(jsonSerializer);
        template.setHashValueSerializer(jsonSerializer);
        template.afterPropertiesSet();
        return template;
    }

    private ObjectMapper objectMapper() {
        ObjectMapper mapper = new ObjectMapper();
        mapper.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
        mapper.activateDefaultTyping(LaissezFaireSubTypeValidator.instance,
                ObjectMapper.DefaultTyping.NON_FINAL,
                JsonTypeInfo.As.PROPERTY);
        return mapper;
    }
}
```

### 4.3 MyBatisPlusConfig — ORM 插件

**职责：** 注册 MyBatis-Plus 插件（分页、乐观锁），配置自动填充处理器。

```java
@Configuration
public class MyBatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        // 分页插件（PostgreSQL 方言）
        interceptor.addInnerInterceptor(
            new PaginationInnerInterceptor(DbType.POSTGRE_SQL));
        // 乐观锁插件（@Version 字段）
        interceptor.addInnerInterceptor(new OptimisticLockerInnerInterceptor());
        return interceptor;
    }
}
```

| 插件 | 说明 |
|------|------|
| `PaginationInnerInterceptor` | 支持 `Page<T>` 分页，自动拼接 `LIMIT x OFFSET y` |
| `OptimisticLockerInnerInterceptor` | 支持 `@Version` 乐观锁，更新时自动校验并递增版本号 |

> **逻辑删除配置：** 在 `application.yml` 的 `mybatis-plus.global-config.db-config` 中统一配置 `logic-delete-field: deleted`，所有 Entity 只需加 `deleted` 字段即可自动生效，无需注解。

### 4.4 AsyncConfig — 异步线程池

**职责：** 提供全局 `@Async` 线程池，覆盖 Spring 默认的单线程异步执行器。

```java
@Configuration
@EnableAsync
public class AsyncConfig implements AsyncConfigurer {

    @Override
    public Executor getAsyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(10);           // 核心线程数
        executor.setMaxPoolSize(50);            // 最大线程数
        executor.setQueueCapacity(100);         // 等待队列
        executor.setKeepAliveSeconds(60);       // 空闲线程回收时间
        executor.setThreadNamePrefix("opencloud-async-");
        executor.setRejectedExecutionHandler(new CallerRunsPolicy()); // 拒绝策略：调用方线程执行
        executor.setWaitForTasksToCompleteOnShutdown(true);  // 优雅停机：等待队列任务完成
        executor.setAwaitTerminationSeconds(30);             // 最长等待 30 秒
        executor.initialize();
        return executor;
    }

    @Override
    public AsyncUncaughtExceptionHandler getAsyncUncaughtExceptionHandler() {
        return (ex, method, params) ->
            log.error("[Async] 异步任务异常 method={} params={}", method.getName(), params, ex);
    }
}
```

| 配置项 | 值 | 说明 |
|-------|-----|------|
| 核心线程数 | 10 | 常驻线程，处理正常并发 |
| 最大线程数 | 50 | 突发流量时扩展 |
| 队列容量 | 100 | 防止大量任务积压 |
| 拒绝策略 | CallerRunsPolicy | 队列满时在调用方线程直接执行，不丢任务 |
| 优雅停机 | 等待 30s | 停机时等待已提交任务完成 |

### 4.5 SchedulerConfig — 定时任务线程池

**职责：** 配置多线程调度池，防止多个定时任务互相阻塞。

```java
@Configuration
public class SchedulerConfig implements SchedulingConfigurer {

    @Override
    public void configureTasks(ScheduledTaskRegistrar taskRegistrar) {
        ThreadPoolTaskScheduler scheduler = new ThreadPoolTaskScheduler();
        scheduler.setPoolSize(10);
        scheduler.setThreadNamePrefix("opencloud-sched-");
        scheduler.setWaitForTasksToCompleteOnShutdown(true);
        scheduler.setAwaitTerminationSeconds(30);
        scheduler.initialize();
        taskRegistrar.setTaskScheduler(scheduler);
    }
}
```

### 4.6 SwaggerConfig — API 文档分组

**职责：** 按功能模块分组 API 文档，方便开发联调。

```java
@Configuration
public class SwaggerConfig {

    @Bean
    public GroupedOpenApi authApi() {
        return GroupedOpenApi.builder()
                .group("01-认证授权")
                .pathsToMatch("/auth/**")
                .build();
    }

    @Bean
    public GroupedOpenApi systemApi() {
        return GroupedOpenApi.builder()
                .group("02-系统管理")
                .pathsToMatch("/system/**")
                .build();
    }

    @Bean
    public GroupedOpenApi cloudApi() {
        return GroupedOpenApi.builder()
                .group("03-云平台管理")
                .pathsToMatch("/cloud/**")
                .build();
    }

    @Bean
    public GroupedOpenApi monitorApi() {
        return GroupedOpenApi.builder()
                .group("04-监控告警")
                .pathsToMatch("/monitor/**")
                .build();
    }

    @Bean
    public OpenAPI openApiInfo() {
        return new OpenAPI()
                .info(new Info()
                        .title("OpenCloud 控制台 API")
                        .description("OpenCloud 云计算管理平台接口文档")
                        .version("v1.0.0"))
                .components(new Components()
                        .addSecuritySchemes("Bearer", new SecurityScheme()
                                .type(SecurityScheme.Type.HTTP)
                                .scheme("bearer")
                                .bearerFormat("JWT")));
    }
}
```

### 4.7 GlobalExceptionHandler — 全局异常处理

**职责：** 统一捕获所有模块未处理的异常，封装为标准 `R<Void>` 响应。

```java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    // 参数校验失败（@Valid / @Validated）
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public R<Void> handleValidation(MethodArgumentNotValidException e) { ... }

    // 参数类型/格式错误
    @ExceptionHandler({HttpMessageNotReadableException.class, MethodArgumentTypeMismatchException.class})
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public R<Void> handleBadRequest(Exception e) { ... }

    // 业务异常（BusinessException 基类）
    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<R<Void>> handleBusiness(BusinessException e) { ... }

    // 403 权限不足
    @ExceptionHandler(AccessDeniedException.class)
    @ResponseStatus(HttpStatus.FORBIDDEN)
    public R<Void> handleForbidden(AccessDeniedException e) { ... }

    // 404 路径不存在（关闭 Spring 默认白标错误页后触发）
    @ExceptionHandler(NoHandlerFoundException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public R<Void> handleNotFound(NoHandlerFoundException e) { ... }

    // 兜底：未知异常
    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public R<Void> handleUnknown(Exception e) {
        log.error("[GlobalExceptionHandler] 未知异常", e);
        return R.fail("SYS_0001", "系统内部错误，请联系管理员");
    }
}
```

## 8. 日志设计（logback-spring.xml）

### 8.1 日志格式

```
%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level [%X{traceId}] %logger{36} - %msg%n
```

| 字段 | 说明 |
|------|------|
| 时间 | `yyyy-MM-dd HH:mm:ss.SSS` |
| 线程 | `[%thread]` |
| 级别 | `%-5level`（左对齐，5位宽） |
| traceId | `[%X{traceId}]`，从 MDC 读取，关联请求链路 |
| 类名 | `%logger{36}`，最多 36 字符 |
| 消息 | `%msg` |

### 8.2 日志 Appender 配置

| Appender | 类型 | 说明 |
|---------|------|------|
| CONSOLE | ConsoleAppender | 开发环境控制台输出，彩色高亮 |
| FILE_INFO | RollingFileAppender | 生产环境 INFO+ 日志，按日滚动，保留 30 天，单文件最大 100MB |
| FILE_ERROR | RollingFileAppender | 生产环境 ERROR 级别单独归档，保留 90 天 |

### 8.3 敏感信息脱敏

在 `RequestLoggingFilter` 中，以下路径不记录请求体：
- `POST /auth/login`（防止密码泄露）
- `POST /auth/password/change`（防止密码泄露）
- `POST /cloud/*/cluster`（防止 AK/SK 泄露）

## 9. 容器化部署设计

### 9.1 Dockerfile（多阶段构建）

```dockerfile
# 阶段一：Maven 编译
FROM maven:3.9-eclipse-temurin-17 AS builder
WORKDIR /workspace
COPY pom.xml .
COPY opencloud-common/pom.xml opencloud-common/
# ... 其他子模块 pom
RUN mvn dependency:resolve -B  # 预下载依赖（利用 Docker Layer 缓存）
COPY . .
RUN mvn package -DskipTests -pl opencloud-web -am

# 阶段二：运行时镜像（精简 JRE）
FROM eclipse-temurin:17-jre
WORKDIR /app
COPY --from=builder /workspace/opencloud-web/target/opencloud-web-*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", \
  "-XX:+UseContainerSupport", \
  "-XX:MaxRAMPercentage=75.0", \
  "-Dspring.profiles.active=${SPRING_PROFILES_ACTIVE:prod}", \
  "-jar", "app.jar"]
```

| 配置项 | 开发环境 | 生产环境 |
|-------|---------|---------|
| JWT_SECRET | 内置默认值 | 必须通过环境变量注入强随机密钥（≥32位） |
| ENCRYPT_KEY | 内置默认值 | 必须通过环境变量注入（32字符） |
| Swagger | 开启 | 关闭（`knife4j.enable=false`） |
| CORS 允许来源 | `*` | 明确域名（`ALLOWED_ORIGINS` 环境变量） |
| Actuator 端点 | 全开放 | 仅 `health` + `prometheus`，且限制 IP |
| SQL 打印 | 开启 | 关闭 |
| 验证码 | 关闭 | 开启 |

## 11. 与其他模块的交互

| 交互方向 | 说明 |
|----------|------|
| web → auth | 装配 Spring Security 过滤链（SecurityConfig Bean）；auth 模块提供 JWT 过滤器和认证相关 Bean |
| web → system | 装配用户/角色/菜单/日志 Controller 和 Service |
| web → cloud | 装配云平台资源管理 Controller 和 Service，CloudClientFactory 工厂 Bean |
| web → monitor | 装配告警规则/记录/通知/WebSocket 相关 Bean |
| web → common | 使用统一响应体 `R<T>`、公共异常基类 `BusinessException`、工具类 |
