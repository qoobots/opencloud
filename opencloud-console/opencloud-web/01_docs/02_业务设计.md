# opencloud-web 业务设计文档

> 模块定位：主应用入口模块，作为整个 OpenCloud 控制台平台的 Spring Boot 启动引导层，负责整合所有子模块（auth/system/cloud/monitor），提供统一的配置管理、全局拦截、健康检查及部署支持能力。
>
> 版本：v1.0 | 最后更新：2026-03-31

---

## 1. 业务背景

`opencloud-web` 是 OpenCloud 控制台的 **主应用启动模块**，承担以下核心职责：

1. **聚合启动**：将 auth、system、cloud、monitor 四个子模块统一整合为一个可运行的 Spring Boot 应用
2. **统一配置**：提供全局 Bean 配置（Web MVC、Redis 序列化、MyBatis-Plus、异步线程池、定时任务、Swagger 文档）
3. **数据库管理**：维护全局初始化 SQL 和 Flyway 版本迁移脚本，统一管理所有模块的 DDL 变更
4. **环境适配**：通过多 Profile 配置（dev/prod/test）支持不同部署环境的差异化参数
5. **运维支撑**：通过 Spring Boot Actuator 暴露健康检查、JVM 指标、应用信息等运维接口
6. **容器化部署**：提供 Dockerfile 和 docker-compose 配置，支持容器化运行

### 1.1 使用角色

| 角色 | 说明 |
|------|------|
| **运维人员** | 负责应用部署、环境配置、健康检查监控，通过 Actuator 接口获取运行状态 |
| **开发人员** | 通过 Swagger UI 查看所有模块 API 文档，进行联调测试 |
| **数据库管理员** | 执行初始化 SQL 脚本、管理 Flyway 版本迁移 |
| **平台管理员** | 使用整合后的平台进行用户、资源、监控等全功能操作 |

## 2. 业务目标

1. 提供统一、稳定的应用启动入口，聚合所有子模块，确保平台可靠运行
2. 通过多环境配置隔离，保障开发/测试/生产环境配置互不影响
3. 提供完整的数据库初始化与版本迁移能力，保障 Schema 与代码版本一致
4. 暴露标准化运维接口（健康检查、指标采集），支持 Prometheus 监控平台集成
5. 统一 API 文档聚合，提升开发协作与接口对接效率
6. 支持容器化一键部署，降低运维门槛

## 3. 核心业务流程

### 3.1 应用启动流程

**触发条件：** 运维人员执行 `java -jar opencloud-web.jar` 或通过 Docker 容器启动应用。

```
应用启动命令执行
  │
  ├─ Spring Boot 自动配置加载
  │   → 读取 application.yml 主配置
  │   → 合并对应 Profile 配置（application-dev.yml / application-prod.yml）
  │   → 注入环境变量（DB_HOST / REDIS_HOST / JWT_SECRET 等）
  │
  ├─ 数据库连接初始化
  │   → HikariCP 连接池建立（PostgreSQL）
  │   → Flyway 自动执行版本迁移脚本（db/migration/*.sql）
  │   → MyBatis-Plus Mapper 扫描注册
  │
  ├─ Redis 连接初始化
  │   → Lettuce 客户端建立连接
  │   → Jackson2 序列化配置生效
  │
  ├─ 各子模块 Bean 装配
  │   → auth 模块：SecurityConfig、JwtTokenProvider、Filter 等
  │   → system 模块：用户/角色/菜单/日志相关 Service 和 Controller
  │   → cloud 模块：CloudClientFactory、各云平台 Service 和 Controller
  │   → monitor 模块：Prometheus 客户端、WebSocket Handler 等
  │
  ├─ 全局配置 Bean 初始化
  │   → WebMvcConfig（CORS / 拦截器注册）
  │   → AsyncConfig（@Async 线程池）
  │   → SchedulerConfig（@Scheduled 线程池）
  │   → SwaggerConfig（API 文档分组）
  │
  ├─ Actuator 端点激活
  │   → /actuator/health → 数据库 + Redis 健康探针
  │   → /actuator/info  → 版本 / 构建时间信息
  │   → /actuator/prometheus → JVM / HTTP 指标
  │
  └─ 应用启动完成，监听 8080 端口，开始接受请求
```

**业务规则：**
- 启动时 Flyway 自动检测 schema 版本，若有未执行的迁移脚本则自动执行
- 环境变量未配置时使用默认值（仅适用于开发环境）
- 生产环境必须通过环境变量注入 `JWT_SECRET`、`DB_PASS`、`REDIS_PASS`

### 3.2 多环境配置管理流程

**触发条件：** 不同部署环境启动应用，需要差异化配置。

```
读取 application.yml（主配置）
  │
  ├─[spring.profiles.active=dev]
  │   → 读取 application-dev.yml
  │   → 日志级别: DEBUG
  │   → SQL 打印: 开启
  │   → 验证码: 关闭
  │   → Actuator: 全端点开放
  │
  ├─[spring.profiles.active=prod]
  │   → 读取 application-prod.yml
  │   → 日志级别: WARN
  │   → SQL 打印: 关闭
  │   → 验证码: 开启
  │   → 连接池: 调优（max-pool-size: 50）
  │   → Actuator: 仅暴露 health + prometheus
  │
  └─[spring.profiles.active=test]
      → 读取 application-test.yml
      → 使用测试数据库配置
      → H2 内存数据库 或 独立测试 PG 实例
```

**业务规则：**
- Profile 激活通过环境变量 `SPRING_PROFILES_ACTIVE` 注入，不硬编码
- 生产配置中所有密钥类参数必须通过环境变量传入，禁止明文写入配置文件

### 3.3 数据库版本迁移流程（Flyway）

**触发条件：** 应用启动时，或部署新版本时。

```
应用启动
  │
  ├─ Flyway 读取 classpath:db/migration/ 下所有脚本
  │
  ├─ 查询 flyway_schema_history 表（首次运行则创建）
  │
  ├─ 对比已执行版本 vs 当前脚本版本
  │   → 未执行的版本脚本按序号升序执行
  │   → 已执行的版本校验 checksum，如不匹配则启动失败（防止脚本篡改）
  │
  └─ 所有脚本执行完毕，标记版本号，应用继续启动
```

**业务规则：**
- 迁移脚本命名：`V{major}.{minor}.{patch}__{描述}.sql`，如 `V1.0.0__init.sql`
- 已执行脚本禁止修改（会触发 Flyway checksum 校验失败）
- 每次新增表/字段/索引变更，需新增一个版本号脚本文件

---

### 3.4 健康检查与运维监控流程

**触发条件：** 外部监控系统（如 K8s liveness probe、Prometheus）周期性探测。

```
外部监控系统发起请求
  │
  ├─ GET /actuator/health
  │   ├─ 检查 PostgreSQL 连接（DataSourceHealthIndicator）
  │   ├─ 检查 Redis 连接（RedisHealthIndicator）
  │   └─ 返回 UP / DOWN 状态
  │
  ├─ GET /actuator/info
  │   → 返回应用名、版本、构建时间（从 build-info.properties 读取）
  │
  └─ GET /actuator/prometheus
      → 返回 Prometheus 文本格式指标
      → 包含：JVM 内存/GC/线程、HTTP 请求数/响应时间、数据库连接池状态
```

**业务规则：**
- `/actuator/prometheus` 在生产环境应限制访问 IP（Nginx ACL 或 Spring Security 限制）
- 健康检查端点（`/actuator/health`）对公网开放，但仅返回 `status` 字段，不暴露详细信息（`show-details: never`）

### 3.6 跨域请求处理流程（CORS）

**触发条件：** 前端 Vue 应用（如运行在 `http://localhost:5173`）向后端发起跨域请求。

```
浏览器发起 OPTIONS 预检请求（跨域场景）
  │
  ├─ WebMvcConfig.addCorsMappings() 匹配规则
  │   → 允许来源：配置的前端域名列表（开发环境允许 localhost:*）
  │   → 允许方法：GET / POST / PUT / DELETE / PATCH / OPTIONS
  │   → 允许 Header：Content-Type / Authorization / X-User-Id / X-Old-Jti
  │   → 暴露 Header：Authorization
  │   → 允许携带 Cookie：false（JWT 无状态，不依赖 Cookie）
  │   → max-age: 3600（预检缓存 1 小时）
  │
  └─ 返回 200 OK 预检响应，浏览器继续发出正式请求
```

**业务规则：**
- 开发环境（dev profile）允许所有来源，方便联调
- 生产环境必须配置明确的 `ALLOWED_ORIGINS` 环境变量，禁止 `*` 通配

---

### 3.7 全局异常处理流程

**触发条件：** 任意业务接口抛出异常，未在各模块内部捕获时。

```
Controller 方法执行抛出异常
  │
  ├─ GlobalExceptionHandler（@RestControllerAdvice）捕获
  │
  ├─ 异常类型分发：
  │   ├─ AuthException         → HTTP 对应状态码 + AUTH_XXXX 错误码
  │   ├─ BusinessException     → HTTP 400 + BIZ_XXXX 错误码
  │   ├─ MethodArgumentNotValidException → HTTP 400 + 字段校验错误列表
  │   ├─ AccessDeniedException → HTTP 403 + AUTH_2001
  │   ├─ AuthenticationException → HTTP 401 + AUTH_1001
  │   └─ 其他 Exception        → HTTP 500 + SYS_0001（内部服务器错误）
  │
  └─ 统一封装为 Result<Void>{ code, message, data: null } 返回
```

---

### 3.8 请求日志记录流程

**触发条件：** 所有进入系统的 HTTP 请求。

```
HTTP 请求到达
  │
  ├─ RequestLoggingFilter（Servlet Filter）
  │   → 记录：请求方法、URL、查询参数、Header（脱敏）、IP
  │   → 生成 traceId（UUID），存入 MDC 上下文，贯穿整个请求链路
  │
  ├─ 业务逻辑执行...
  │
  ├─ 响应返回时记录：
  │   → 响应状态码、耗时（ms）
  │   → 敏感路径（如 /auth/login）不记录请求体（防密码泄露）
  │
  └─ 清理 MDC，释放资源
```

---

### 3.9 优雅停机流程

**触发条件：** 运维人员执行 `kill -15`、`docker stop` 或 K8s 滚动更新触发 Pod 终止。

```
接收 SIGTERM 信号
  │
  ├─ Spring Boot Graceful Shutdown 启动
  │   → 拒绝新请求（503 Service Unavailable）
  │   → 等待正在处理的请求完成（超时 30 秒）
  │
  ├─ @PreDestroy 回调
  │   → 定时任务线程池：停止接受新任务，等待当前任务执行完
  │   → 异步线程池：优雅关闭，等待队列消费完
  │   → WebSocket 连接：向客户端发送关闭帧，断开所有连接
  │
  ├─ Spring 容器关闭（DestroyBean 逐一回调）
  │   → 数据库连接池 HikariCP 释放所有连接
  │   → Redis Lettuce 客户端断开连接
  │
  └─ 进程退出，exit code = 0
```

**业务规则：**
- 优雅停机超时 30 秒，超时后强制终止
- K8s 滚动更新场景中，需配置 `terminationGracePeriodSeconds: 60`

## 4. 功能模块详述

### 4.1 应用启动与主配置

| 功能点 | 业务描述 | 关键规则 |
|-------|---------|---------|
| **主启动类** | `OpenCloudApplication` 作为 Spring Boot 入口，启用 `@MapperScan`、`@EnableAsync`、`@EnableScheduling` | 包扫描路径覆盖所有子模块 |
| **主配置文件** | `application.yml` 汇总数据源、Redis、MyBatis-Plus、JWT、云平台、日志等全局配置 | 敏感参数通过环境变量注入 |
| **多环境配置** | dev/prod/test 三套配置，通过 `spring.profiles.active` 激活 | 生产环境强制关闭 SQL 打印、开启验证码 |

### 4.2 全局 Bean 配置

| 配置类 | 业务描述 | 关键规则 |
|-------|---------|---------|
| **WebMvcConfig** | 配置全局 CORS 跨域策略、静态资源映射、全局拦截器注册 | 生产环境需明确配置允许来源 |
| **RedisConfig** | Jackson2 序列化、Key 过期事件监听配置 | 序列化使用 JSON，方便 Redis 可读性调试 |
| **MyBatisPlusConfig** | 分页插件（PaginationInnerInterceptor）、乐观锁插件、逻辑删除自动处理 | 分页查询必须使用 Page 对象 |
| **AsyncConfig** | `@Async` 全局异步线程池：核心 10 线程、最大 50 线程、队列 100、拒绝策略 CallerRunsPolicy | 线程名前缀 `opencloud-async-` |
| **SchedulerConfig** | `@Scheduled` 定时任务线程池：10 线程，防止单线程阻塞 | 线程名前缀 `opencloud-sched-` |
| **SwaggerConfig** | 按模块分组（auth/system/cloud/monitor），集成 Knife4j UI | 生产环境可通过配置关闭 Swagger |

### 4.3 数据库管理

| 功能点 | 业务描述 | 关键规则 |
|-------|---------|---------|
| **初始化脚本** | `db/init.sql` 包含全量 DDL（所有模块建表语句）+ 初始数据（超管账号、角色） | 包含系统表、云平台表、监控表 |
| **Flyway 迁移** | `db/migration/` 目录下按版本号管理数据库变更脚本 | 已执行脚本禁止修改 |
| **初始数据** | 超级管理员账号（admin/Admin@123）、默认角色（ROLE_ADMIN/ROLE_VIEWER）、基础菜单树 | 密码 BCrypt 加密存储 |

### 4.4 健康检查与运维

| 功能点 | 接口 | 业务描述 | 关键规则 |
|-------|------|---------|---------|
| **健康检查** | `GET /actuator/health` | 返回数据库+Redis 健康状态 | 生产环境仅暴露 status，不暴露详情 |
| **应用信息** | `GET /actuator/info` | 返回版本号、构建时间 | 通过 Maven 插件生成 build-info |
| **Prometheus 指标** | `GET /actuator/prometheus` | JVM/HTTP/DB 连接池指标 | 生产环境限制 IP 访问 |
| **指标详情** | `GET /actuator/metrics` | 单个指标详情查询 | 仅限内部网络 |

### 4.5 打包与容器化部署

| 功能点 | 业务描述 | 关键规则 |
|-------|---------|---------|
| **Dockerfile** | 多阶段构建：Maven 编译阶段 + JRE 17 精简运行阶段 | 基础镜像使用 `eclipse-temurin:17-jre` |
| **docker-compose** | 开发环境一键启动：应用 + PostgreSQL + Redis + Prometheus + Grafana + AlertManager | 通过 `.env` 文件注入环境变量 |
| **.env.example** | 环境变量示例文件，包含所有必填变量及说明 | 正式部署时复制为 `.env` 并填写真实值 |
| **logback-spring.xml** | 日志配置：控制台输出 + 滚动文件归档（按日切割，保留 30 天） | 生产环境日志文件输出到 `/logs/opencloud/` |

### 4.6 优雅停机与生命周期管理

| 功能点 | 业务描述 | 关键规则 |
|-------|---------|---------|
| **优雅停机** | 接收 SIGTERM 后停止接受新请求，等待现有请求完成后关闭 | 超时时间 30 秒，超时后强制终止 |
| **线程池优雅关闭** | 异步线程池和定时任务线程池在关闭前等待队列任务执行完毕 | 防止任务中途中断，数据不一致 |
| **WebSocket 优雅断开** | 停机时向所有 WebSocket 客户端发送关闭帧 | 客户端收到关闭帧后触发重连逻辑 |

### 4.7 全局异常处理

| 功能点 | 业务描述 | 关键规则 |
|-------|---------|---------|
| **全局异常兜底** | 捕获所有未被子模块处理的异常，统一封装为 `R<Void>` 响应 | 异常详情仅记录日志，不对外暴露堆栈 |
| **参数校验失败** | `@Valid` / `@Validated` 校验不通过时，返回字段错误列表 | HTTP 400，格式：`{field, message}` 列表 |
| **404 处理** | 请求路径不存在时，返回统一 404 响应（而非 Spring 默认白标错误页） | 返回 JSON 格式，不返回 HTML |

## 5. 业务规则汇总

### 5.1 环境变量规则

| 变量名 | 说明 | 是否必填（生产） | 默认值（开发） |
|--------|------|:---:|------|
| `DB_HOST` | 数据库主机 | ✅ | `localhost` |
| `DB_PORT` | 数据库端口 | ✅ | `5432` |
| `DB_NAME` | 数据库名 | ✅ | `opencloud` |
| `DB_USER` | 数据库用户名 | ✅ | `opencloud` |
| `DB_PASS` | 数据库密码 | ✅ | `opencloud123` |
| `REDIS_HOST` | Redis 主机 | ✅ | `localhost` |
| `REDIS_PORT` | Redis 端口 | ✅ | `6379` |
| `REDIS_PASS` | Redis 密码 | 推荐 | 空 |
| `JWT_SECRET` | JWT 签名密钥（≥32 位随机字符串） | ✅ | 内置默认值（禁用于生产） |
| `JWT_ACCESS_TTL` | Access Token 有效期（秒） | — | `7200`（2 小时） |
| `JWT_REFRESH_TTL` | Refresh Token 有效期（秒） | — | `604800`（7 天） |
| `PROMETHEUS_URL` | Prometheus 地址 | ✅ | `http://prometheus:9090` |
| `GRAFANA_URL` | Grafana 地址 | ✅ | `http://grafana:3000` |
| `GRAFANA_TOKEN` | Grafana Service Account Token | ✅ | 空 |
| `MAIL_HOST` | SMTP 服务器 | 按需 | `smtp.example.com` |
| `ALLOWED_ORIGINS` | 允许的前端域名（逗号分隔） | ✅ | `*`（开发） |
| `ENCRYPT_KEY` | AES-256-GCM 加密密钥（云平台/通知渠道敏感配置） | ✅ | 内置默认（禁用于生产） |

### 5.2 Flyway 迁移规则

| 规则 | 说明 |
|------|------|
| 命名规范 | `V{major}.{minor}.{patch}__{description}.sql`，如 `V1.0.0__init.sql` |
| 禁止修改 | 已在 flyway_schema_history 中执行的脚本禁止修改，会触发 checksum 校验失败 |
| 回滚策略 | 不支持自动回滚；回滚需手动编写补偿 SQL，以 `V{新版本}__rollback_{旧版本}.sql` 命名 |
| 测试环境 | 测试环境可配置 `spring.flyway.clean-on-validation-error=true`，但生产严禁 |

### 5.3 日志规则

| 规则 | 说明 |
|------|------|
| 日志格式 | `[时间] [级别] [traceId] [线程] 类名 - 日志内容` |
| 敏感信息脱敏 | 密码、Token、密钥等字段在请求日志中脱敏（以 `****` 替代） |
| 日志保留 | 生产环境日志文件保留 30 天，按日滚动 |
| 请求日志 | 所有 HTTP 请求记录 method/url/status/costTime/traceId |

### 5.4 启动安全检查规则

| 规则 | 说明 |
|------|------|
| JWT_SECRET 强度 | 生产环境检测 JWT_SECRET 长度 < 32 位时，启动时打印 **WARN** 日志 |
| ENCRYPT_KEY 检测 | 使用内置默认 ENCRYPT_KEY 时，生产 profile 下触发启动 **WARN** 日志 |
| Swagger 生产关闭 | 生产环境 `knife4j.enable=false`，不对外暴露 API 文档 |

## 6. 业务异常场景

### 6.1 启动失败场景

| 场景 | 处理策略 |
|------|---------|
| 数据库连接失败 | 应用启动失败，打印连接拒绝日志，检查 DB_HOST/DB_PASS 配置 |
| Redis 连接失败 | 应用启动失败，打印连接拒绝日志，检查 REDIS_HOST 配置 |
| Flyway 校验失败 | 应用启动失败，打印 checksum 不匹配的脚本名称，禁止启动防止数据损坏 |
| Flyway 脚本执行失败 | 应用启动失败，事务回滚，不写入 flyway_schema_history，打印具体 SQL 错误 |
| 端口被占用 | 应用启动失败，提示 `Address already in use: 8080` |

### 6.2 运行时场景

| 场景 | 处理策略 |
|------|---------|
| 数据库连接池耗尽 | 请求等待连接超时（默认 30s），超时后返回 500，HikariCP 自动打印线程状态 |
| Redis 连接断开 | 影响 Token 校验（auth 模块）、权限缓存；Lettuce 自动重连，重连期间相关功能降级 |
| 异步线程池满载 | 使用 CallerRunsPolicy 策略，在调用方线程中直接执行，防止任务丢失 |
| 定时任务线程阻塞 | 多线程调度，单任务阻塞不影响其他定时任务执行 |

## 8. 业务术语表

| 术语 | 说明 |
|-----|------|
| **Profile** | Spring Boot 多环境配置标识（dev/prod/test） |
| **Flyway** | 数据库版本迁移工具，自动管理 SQL 脚本版本 |
| **Actuator** | Spring Boot 内置运维端点，提供健康检查、指标、应用信息等 |
| **Knife4j** | 基于 Swagger 的增强 API 文档 UI 框架 |
| **CORS** | 跨域资源共享，允许浏览器跨域访问 API |
| **traceId** | 请求链路唯一标识，通过 MDC 贯穿整个请求的日志 |
| **HikariCP** | 高性能 JDBC 连接池，Spring Boot 默认选型 |
| **Lettuce** | Redis 客户端，支持响应式编程，Spring Boot Redis 默认实现 |
| **AES-256-GCM** | 对称加密算法，用于加密存储敏感配置（云平台密钥、通知渠道配置） |
