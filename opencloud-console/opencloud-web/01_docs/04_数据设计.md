# opencloud-web 数据设计文档

> 模块定位：opencloud-web 本身不新增业务数据表，但作为统一数据库管理入口，负责通过 Flyway 维护整个平台所有模块的数据库 Schema 版本。本文档汇总全平台所有表的设计与说明，作为唯一数据设计权威文档。
>
> 版本：v1.0 | 最后更新：2026-03-31

---

## 1. 数据库概览

### 1.1 数据库选型

| 项目 | 选择 | 说明 |
|------|------|------|
| 数据库 | PostgreSQL 16 | 关系型数据库，支持 JSONB、GIN 索引、事务完整性 |
| 缓存 | Redis 7 | Token 缓存、权限缓存、验证码、限流计数器 |
| ORM | MyBatis-Plus 3.5.x | 分页、逻辑删除、自动填充 |

### 1.2 表命名规范

| 模块 | 前缀 | 示例 |
|------|------|------|
| 认证授权（auth） | `auth_` | `auth_user_credential`, `auth_login_log` |
| 系统管理（system） | `sys_` | `sys_user`, `sys_role`, `sys_menu` |
| 云平台对接（cloud） | `cloud_` | `cloud_cluster_config`, `cloud_task` |
| 监控告警（monitor） | `monitor_` | `monitor_alert_rule`, `monitor_alert_record` |

### 1.3 通用字段规范

所有业务表（关联表除外）必须包含以下字段：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| `id` | `BIGINT PRIMARY KEY` | 雪花算法生成，全局唯一 |
| `tenant_id` | `BIGINT` | 租户 ID，多租户数据隔离（部分表无需租户隔离时可省略） |
| `created_at` | `TIMESTAMP NOT NULL DEFAULT NOW()` | 创建时间（MyBatis-Plus 自动填充） |
| `updated_at` | `TIMESTAMP NOT NULL DEFAULT NOW()` | 更新时间（MyBatis-Plus 自动填充） |
| `created_by` | `BIGINT` | 创建人 ID（MyBatis-Plus 自动填充，取 SecurityContext 中的 userId） |
| `updated_by` | `BIGINT` | 更新人 ID |
| `deleted` | `SMALLINT NOT NULL DEFAULT 0` | 逻辑删除标志：0=未删除，1=已删除 |

## 2. 认证授权模块（auth）数据表

### 2.1 auth_user_credential — 用户凭证表

**业务用途：** 存储用户登录凭证（密码哈希、账号状态、登录失败计数等），与 `sys_user` 一对一关联。

```sql
CREATE TABLE auth_user_credential (
    id                  BIGINT          PRIMARY KEY,
    user_id             BIGINT          NOT NULL UNIQUE COMMENT '关联 sys_user.id，一对一',
    tenant_id           BIGINT          NOT NULL COMMENT '租户 ID',
    username            VARCHAR(64)     NOT NULL UNIQUE COMMENT '登录账号（冗余存储，登录时免 JOIN）',
    password_hash       VARCHAR(128)    NOT NULL COMMENT 'BCrypt 加密密码',
    status              VARCHAR(16)     NOT NULL DEFAULT 'ACTIVE' COMMENT 'ACTIVE/DISABLED/LOCKED',
    login_fail_count    SMALLINT        NOT NULL DEFAULT 0 COMMENT '连续登录失败次数',
    lock_expire_time    TIMESTAMP       COMMENT '账号锁定到期时间，NULL 表示未锁定',
    last_login_time     TIMESTAMP       COMMENT '上次登录时间',
    password_update_time TIMESTAMP      COMMENT '密码最近修改时间',
    created_at          TIMESTAMP       NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP       NOT NULL DEFAULT NOW(),
    created_by          BIGINT,
    updated_by          BIGINT,
    deleted             SMALLINT        NOT NULL DEFAULT 0
);

CREATE UNIQUE INDEX idx_auth_cred_user_id  ON auth_user_credential(user_id)   WHERE deleted = 0;
CREATE UNIQUE INDEX idx_auth_cred_username ON auth_user_credential(username)  WHERE deleted = 0;
```

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | BIGINT | 关联 sys_user.id，一对一 |
| username | VARCHAR(64) | 登录账号（冗余存储，登录时免 JOIN sys_user） |
| password_hash | VARCHAR(128) | BCrypt(cost=12) 加密 |
| status | VARCHAR(16) | `ACTIVE`/`DISABLED`/`LOCKED` |
| login_fail_count | SMALLINT | 连续登录失败次数，成功后清零 |
| lock_expire_time | TIMESTAMP | 锁定到期时间；达到锁定阈值时设置 `NOW() + 15min` |

### 2.2 auth_login_log — 登录日志表

**业务用途：** 记录每次登录/登出行为，用于安全审计与异常登录检测。

```sql
CREATE TABLE auth_login_log (
    id              BIGINT          PRIMARY KEY,
    user_id         BIGINT          COMMENT '用户 ID（登录失败时可能为 NULL）',
    tenant_id       BIGINT          COMMENT '租户 ID',
    username        VARCHAR(64)     NOT NULL COMMENT '登录账号',
    login_ip        VARCHAR(64)     COMMENT '登录 IP（支持 IPv6）',
    user_agent      VARCHAR(512)    COMMENT '浏览器/客户端 User-Agent',
    login_time      TIMESTAMP       NOT NULL DEFAULT NOW() COMMENT '登录时间',
    logout_time     TIMESTAMP       COMMENT '登出时间（登出时补填）',
    status          VARCHAR(16)     NOT NULL COMMENT 'SUCCESS/FAILED/LOGOUT/FORCE_LOGOUT',
    fail_reason     VARCHAR(128)    COMMENT '失败原因（PASSWORD_ERROR/ACCOUNT_LOCKED 等）',
    created_at      TIMESTAMP       NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_auth_login_log_user_id    ON auth_login_log(user_id);
CREATE INDEX idx_auth_login_log_username   ON auth_login_log(username);
CREATE INDEX idx_auth_login_log_login_time ON auth_login_log(login_time DESC);
CREATE INDEX idx_auth_login_log_status     ON auth_login_log(status);
```

```sql
    remark          VARCHAR(256)    COMMENT '备注',
    created_at      TIMESTAMP       NOT NULL DEFAULT NOW()

CREATE INDEX idx_auth_op_log_operator_id  ON auth_operation_log(operator_id);

```

## 3. 系统管理模块（system）数据表

### 3.1 sys_user — 系统用户表

**业务用途：** 存储用户基础信息（昵称、邮箱、头像等），认证信息由 `auth_user_credential` 单独管理。

```sql
CREATE TABLE sys_user (
    id          BIGINT          PRIMARY KEY,
    tenant_id   BIGINT          NOT NULL COMMENT '所属租户',
    username    VARCHAR(64)     NOT NULL COMMENT '登录账号',
    nickname    VARCHAR(64)     COMMENT '显示名称',
    email       VARCHAR(128)    COMMENT '邮箱',
    phone       VARCHAR(20)     COMMENT '手机号',
    avatar      VARCHAR(512)    COMMENT '头像 URL',
    gender      SMALLINT        NOT NULL DEFAULT 0 COMMENT '0=未知 1=男 2=女',
    status      SMALLINT        NOT NULL DEFAULT 0 COMMENT '0=正常 1=禁用',
    remark      VARCHAR(512)    COMMENT '备注',
    created_at  TIMESTAMP       NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMP       NOT NULL DEFAULT NOW(),
    created_by  BIGINT,
    updated_by  BIGINT,
    deleted     SMALLINT        NOT NULL DEFAULT 0
);

CREATE UNIQUE INDEX idx_sys_user_username  ON sys_user(username, tenant_id) WHERE deleted = 0;
CREATE INDEX        idx_sys_user_tenant_id ON sys_user(tenant_id) WHERE deleted = 0;
```

### 3.6 sys_operation_log — 操作日志表

**业务用途：** AOP 自动记录所有有 `@Log` 注解的接口调用，用于操作审计。

```sql
    error_msg       TEXT            COMMENT '异常信息（失败时记录）',
```

## 6. Redis Key 设计（汇总）

### 6.1 认证授权（auth 模块）

| Key 模式 | 值类型 | TTL | 说明 |
|---------|------|-----|------|
| `auth:captcha:{captchaKey}` | String | 5 分钟 | 验证码答案（统一转小写存储） |
| `auth:refresh:{userId}:{jti}` | String | 7 天 | RefreshToken 详情 JSON |
| `auth:blacklist:{jti}` | String | Token 剩余有效期 | AccessToken 黑名单标记，值为 "1" |
| `auth:perm:{userId}` | String(JSON) | 10 分钟 | 用户权限标识集合缓存 |
| `auth:token:version:{userId}` | String | 永久 | Token 版本号，强制下线时 +1 |
| `auth:login:rate:{ip}` | String | 1 分钟 | 登录请求限频计数器（IP 维度） |

## 9. 数据安全设计

| 场景 | 方案 |
|------|------|
| 用户密码 | BCrypt(cost=12) 单向加密存储，永不解密 |
| 云平台 AK/SK、Token | AES-256-GCM 加密存储（`cloud_cluster_config.config_json`），密钥通过 `ENCRYPT_KEY` 环境变量注入 |
| 通知渠道 Webhook URL/Token | AES-256-GCM 加密存储（`monitor_notify_channel.config_json`） |
| API 响应脱敏 | 上述加密字段在 GET 接口中以 `****` 掩码返回，不解密暴露 |
| 操作日志请求参数 | 密码字段（`password`, `oldPassword`, `newPassword`）在记录前替换为 `****` |
