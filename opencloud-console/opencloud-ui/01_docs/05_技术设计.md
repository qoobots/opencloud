## 二、工程化配置

### 2.1 Vite 插件配置

```typescript
// vite.config.ts 核心插件
plugins: [
  vue(),
  // 自动导入 Vue/Pinia API (ref, computed 等)，无需手动 import
  AutoImport({
    imports: ['vue', 'vue-router', 'pinia'],
    resolvers: [ElementPlusResolver()],
    dts: 'src/auto-imports.d.ts',
  }),
  // 自动按需注册 Element Plus 组件
  Components({
    resolvers: [ElementPlusResolver()],
    dts: 'src/components.d.ts',
  }),
]
```

### 2.2 TypeScript 配置

`tsconfig.json` 严格模式配置：
- `strict: true`
- `noImplicitAny: true`
- `baseUrl: "."`
- `paths: { "@/*": ["src/*"] }`

### 2.3 ESLint 规范

采用 Vue 官方推荐规则：
- `plugin:vue/vue3-recommended`
- `@vue/eslint-config-typescript`
- 强制组件名多单词（`vue/multi-word-component-names`）
- 禁用 `any` 警告（在业务初期可适当放宽，后续收紧）

## 三、编码规范

### 3.1 Vue 组件规范

1. **采用 `<script setup lang="ts">` 语法**。
2. **组件命名**：PascalCase（如 `MetricCard.vue`），视图组件以 `View` 结尾（如 `LoginView.vue`）。
3. **模板结构**：
   ```vue
   <template>
     <div class="component-name">...</div>
   </template>

   <script setup lang="ts">
   // 1. imports
   // 2. props & emits
   // 3. refs & reactive
   // 4. computed
   // 5. methods
   // 6. lifecycle hooks
   </script>

   <style scoped>
   /* 样式必须 scoped */
   </style>
   ```

### 3.2 CSS 规范

1. **优先使用 Element Plus CSS 变量**（如 `var(--el-color-primary)`）保持主题一致性。
2. 全局样式写在 `src/styles/index.css`。
3. 避免使用深度选择器 `:deep()`，除非必须覆盖 Element Plus 内部样式。

### 3.3 API 封装规范

1. 每个业务模块对应一个 API 文件（如 `src/api/cloud.ts`）。
2. 接口参数和返回值必须定义 TypeScript 接口。
3. 统一使用封装好的 `request` 实例。

```typescript
import request from '@/utils/request'
import type { PageQuery, PageResult } from './types'

export interface InstanceVO { /* ... */ }

export function getInstanceList(params: PageQuery) {
  return request.get<any, PageResult<InstanceVO>>('/cloud/openstack/instances', { params })
}
```

## 四、性能优化策略

1. **路由懒加载**：所有视图组件使用 `() => import('@/views/...')` 动态导入。
2. **组件按需加载**：通过 `unplugin-vue-components` 实现 Element Plus 的按需打包。
3. **分包构建**：在 `vite.config.ts` 中配置 `manualChunks`，将 `vue`、`element-plus`、`echarts` 等大体积依赖单独打包，利用浏览器长效缓存。
4. **图表按需引入**：ECharts 仅引入使用的图表类型和组件，减小打包体积。
5. **DOM 节点复用**：长列表使用分页，避免一次性渲染过多 DOM。

## 五、部署方案

### 5.1 Docker 部署

使用多阶段构建（Multi-stage Build），基于 Nginx 提供静态文件服务。

```dockerfile
# 阶段 1：构建
FROM node:20-alpine as builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# 阶段 2：运行
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 5.2 Nginx 配置要点

1. **History 路由支持**：`try_files $uri $uri/ /index.html;`
2. **API 反向代理**：将 `/api/` 转发到后端服务。
3. **WebSocket 代理**：支持 `Upgrade` 和 `Connection` 头。
4. **Gzip 压缩**：开启静态资源压缩。

前端技术选型与总体技术设计文档 (`01_docs/05_总体技术设计.md`) 完全对齐：

| 类别 | 总体技术设计 | 前端实现 | 一致性 |
|------|-------------|----------|--------|
| **前端框架** | Vue 3.x | ✅ Vue ^3.4.x | ✅ |
| **构建工具** | Vite 5.x | ✅ Vite ^5.2.x | ✅ |
| **UI 组件** | Element Plus 2.x | ✅ Element Plus ^2.7.x | ✅ |
| **状态管理** | Pinia 2.x | ✅ Pinia ^2.1.x | ✅ |
| **HTTP 客户端** | Axios 1.x | ✅ Axios ^1.6.x | ✅ |
| **图表库** | ECharts 5.x | ✅ ECharts ^5.5.x | ✅ |
| **路由** | Vue Router 4.x | ✅ Vue Router ^4.3.x | ✅ |

### 6.2 后端技术对应

前端需与后端特定技术实现配合：

| 后端技术 | 前端对应实现 | 说明 |
|---------|-------------|------|
| Spring Boot 3.2.x | Vite 开发服务器 | 通过代理对接后端 8080 端口 |
| MyBatis-Plus | 列表分页组件 | 自动处理 pageNum/pageSize 参数 |
| JWT 双 Token | Axios 拦截器 | 自动附加 Authorization 头 |
| Redis 会话 | 前端无感知 | Token 存储在 localStorage |
| WebSocket | useWebSocket Hook | 实时告警推送 |
| AES-256-GCM | 前端无感知 | 敏感数据在后端处理，前端只展示脱敏值 |

### 6.3 安全技术对应

| 后端安全机制 | 前端配合实现 |
|-------------|-------------|
| JWT 认证 | 请求拦截器自动附加 Token |
| Token 黑名单 | 登出时清除本地 Token |
| 密码强度校验 | 表单实时显示密码强度 |
| 敏感数据加密 | 展示脱敏后的值 |
| 操作日志 AOP | 前端无感知（后端自动记录） |
