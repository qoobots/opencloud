## 一、应用架构

`opencloud-ui` 采用基于 Vue 3 Composition API 的单页应用（SPA）架构。

### 1.1 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                      视图层 (Views)                          │
│  • 页面组件 (src/views/*)                                   │
│  • 布局组件 (src/layouts/*)                                 │
├─────────────────────────────────────────────────────────────┤
│                      组件层 (Components)                     │
│  • 业务组件 (业务特有，复用性低)                             │
│  • 基础组件 (src/components/*，如 StatusBadge, MetricCard)  │
├─────────────────────────────────────────────────────────────┤
│                      状态与路由层                            │
│  • Pinia Stores (src/stores/*)                              │
│  • Vue Router (src/router/*)                                │
├─────────────────────────────────────────────────────────────┤
│                      服务与通信层                            │
│  • API 封装 (src/api/*)                                     │
│  • Axios 拦截器 (src/utils/request.ts)                      │
│  • WebSocket 客户端 (src/hooks/useWebSocket.ts)             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 目录结构规范

```
src/
├── api/            # 接口请求封装，按业务模块划分 (auth.ts, cloud.ts 等)
├── assets/         # 静态资源 (图片、SVG 图标等)
├── components/     # 全局公共组件
├── directives/     # 全局自定义指令 (如 v-permission)
├── hooks/          # 组合式函数 (Composables)
├── layouts/        # 页面布局组件 (MainLayout.vue 等)
├── router/         # 路由配置与守卫
├── stores/         # Pinia 状态管理
├── styles/         # 全局样式 (index.css, variables.css)
├── utils/          # 工具函数 (request.ts, format.ts 等)
├── views/          # 页面级组件，按业务模块划分目录
├── App.vue         # 根组件
└── main.ts         # 入口文件
```

## 二、核心组件设计

### 2.1 路由与布局设计

- **MainLayout**：包含 Sidebar（侧边栏）、Header（顶栏面包屑+用户信息）、AppMain（路由出口 `<router-view>`）。
- **路由配置**：
  - 静态路由：`/login`, `/404`, `/403`。
  - 动态路由：基于后端的菜单数据动态挂载到 `MainLayout` 下。
- **路由守卫**：
  - `beforeEach`：校验 Token、校验页面权限、开启 NProgress。
  - `afterEach`：关闭 NProgress、设置 `document.title`。

### 2.2 状态管理 (Pinia)

- **userStore** (`src/stores/user.ts`)：
  - 状态：`accessToken`, `refreshToken`, `userInfo`。
  - Getter：`isLoggedIn`, `roles`, `permissions`。
  - Action：`login`, `logout`, `tryRefreshToken`, `hasPermission`。
- **alertStore** (`src/stores/alert.ts`)：
  - 状态：`activeAlerts` (当前活跃告警列表), `unreadCount`。
  - Action：`addAlert`, `removeAlert`, `clearAll`。

### 2.3 HTTP 请求封装 (Axios)

`src/utils/request.ts` 核心逻辑：
1. **请求拦截**：自动在 Header 中附加 `Authorization: Bearer ${token}`。
2. **响应拦截**：
   - 剥离外层包装，直接返回 `res.data.data`（成功时）。
   - 统一处理业务错误码（`res.data.code !== 200`），弹出 `ElMessage.error`。
3. **401 无感刷新**：
   - 捕获 401 状态码。
   - 触发 `userStore.tryRefreshToken()`。
   - 刷新成功则重试原请求；失败则跳转 `/login`。

### 2.4 权限控制设计

1. **路由级权限**：在路由 `meta.permission` 中定义，路由守卫拦截。
2. **按钮级权限**：自定义指令 `v-permission`。
   ```vue
   <!-- 只有拥有 system:user:create 权限的用户才能看到此按钮 -->
   <el-button v-permission="'system:user:create'">新增用户</el-button>
   ```
   实现原理：在 `mounted` 钩子中，调用 `userStore.hasPermission`，若无权限则从 DOM 中移除该元素（`el.parentNode?.removeChild(el)`）。

## 三、公共 Composable 设计

### 3.1 `useTable` (表格通用逻辑)

封装分页、加载状态、查询等通用逻辑，减少样板代码。

```typescript
export function useTable<T>(apiMethod: Function, initQuery = {}) {
  const loading = ref(false)
  const tableData = ref<T[]>([])
  const total = ref(0)
  const queryParams = reactive({ pageNum: 1, pageSize: 10, ...initQuery })

  const getList = async () => {
    loading.value = true
    try {
      const res = await apiMethod(queryParams)
      tableData.value = res.list
      total.value = res.total
    } finally {
      loading.value = false
    }
  }

  // 监听分页变化自动请求
  watch(() => [queryParams.pageNum, queryParams.pageSize], getList)

  return { loading, tableData, total, queryParams, getList }
}
```

## 四、构建与优化

### 4.1 Vite 配置优化

- **自动导入**：使用 `unplugin-auto-import` 自动导入 Vue API（ref, reactive 等）和 Vue Router/Pinia API。
- **组件按需加载**：使用 `unplugin-vue-components` 自动按需导入 Element Plus 组件及样式。
- **分包策略 (Manual Chunks)**：
  - `vendor`：vue, vue-router, pinia
  - `elementPlus`：element-plus
  - `echarts`：echarts, vue-echarts
  - 业务代码按路由懒加载（`() => import('...')`）。

### 4.2 环境变量

- `.env.development`：开发环境，配置代理等。
- `.env.production`：生产环境，配置线上 API 基础路径。

## 五、前后端集成方案

### 5.1 API 对接规范

前端与后端通过 RESTful API 通信，采用统一的响应封装格式：

```typescript
// 统一响应格式 R<T>
interface ApiResponse<T> {
  code: number;      // 业务状态码 (200=成功)
  message: string;   // 提示信息
  data: T;          // 响应数据
  timestamp: number;
}

// 分页响应格式
interface PageResponse<T> {
  list: T[];
  total: number;
  pageNum: number;
  pageSize: number;
  pages: number;
}
```

**HTTP 状态码与业务码映射：**

| HTTP 状态码 | 业务码 | 含义 | 前端处理 |
|------------|--------|------|----------|
| 200 | 200 | 成功 | 直接返回 data |
| 400 | 400 | 请求参数错误 | ElMessage.error 提示 |
| 401 | 401 | 未认证/Token 过期 | 尝试刷新 Token |
| 403 | 403 | 无权限 | ElMessage.error 提示无权限 |
| 404 | 404 | 资源不存在 | ElMessage.error 提示 |
| 423 | 423 | 账号锁定 | 提示剩余锁定时间 |
| 429 | 429 | 请求过于频繁 | 提示稍后重试 |
| 500 | 500 | 服务器内部错误 | ElMessage.error 提示 |

### 5.2 请求/响应拦截器

```typescript
// src/utils/request.ts

// 请求拦截器
request.interceptors.request.use(
  (config) => {
    // 自动附加 Token
    const token = userStore.accessToken
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    // 附加租户 ID（多租户场景）
    if (userStore.userInfo?.tenantId) {
      config.headers['X-Tenant-ID'] = userStore.userInfo.tenantId
    }
    return config
  },
  (error) => Promise.reject(error)
)

// 响应拦截器
request.interceptors.response.use(
  (response) => {
    const res = response.data
    if (res.code === 200) {
      return res.data
    }
    // 业务错误
    ElMessage.error(res.message || '操作失败')
    return Promise.reject(new Error(res.message))
  },
  async (error) => {
    if (error.response?.status === 401) {
      // Token 过期，尝试刷新
      if (await userStore.tryRefreshToken()) {
        // 刷新成功，重试原请求
        return request.request(error.config)
      }
    }
    ElMessage.error(error.message || '网络错误')
    return Promise.reject(error)
  }
)
```

### 5.4 Docker 部署配置

生产环境前后端分离部署：

```yaml
# docker-compose.yml (片段)
services:
  # 前端 Nginx
  opencloud-ui:
    image: opencloud-ui:latest
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - opencloud-web

  # 后端 Spring Boot
  opencloud-web:
    image: opencloud-web:latest
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - SPRING_DATASOURCE_URL=jdbc:postgresql://postgres:5432/opencloud
      - SPRING_REDIS_HOST=redis
    depends_on:
      - postgres
      - redis
```

```nginx
# nginx.conf (片段)
server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api {
        proxy_pass http://opencloud-web:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://opencloud-web:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 六、与总体应用设计的对应

### 6.1 技术栈一致性

前端技术选型与总体技术设计保持一致：

| 类别 | 总体技术设计 | 前端实现 |
|------|-------------|----------|
| 框架 | Vue 3 | ✅ Vue ^3.4.x |
| 构建 | Vite 5 | ✅ Vite ^5.2.x |
| UI 组件 | Element Plus | ✅ Element Plus ^2.7.x |
| 状态管理 | Pinia | ✅ Pinia ^2.1.x |
| 图表 | ECharts 5 | ✅ ECharts ^5.5.x |

### 6.2 后端模块对应

前端页面与后端模块的对应关系：

| 后端模块 | 前端页面/功能 |
|---------|--------------|
| opencloud-auth | 登录页、Token 刷新、登出 |
| opencloud-system | 用户管理、角色管理、菜单管理、部门管理、日志查询 |
| opencloud-cloud | 集群管理、云主机管理、网络管理、存储管理、K8s 管理 |
| opencloud-monitor | 监控大盘、告警规则、告警记录、通知渠道、WebSocket |

### 6.3 核心组件对照

| 后端组件 | 前端对应实现 |
|---------|-------------|
| JWT 认证过滤器 | Axios 请求拦截器（Token 附加） |
| 全局异常处理器 | Axios 响应拦截器（错误处理） |
| WebSocket 处理器 | useWebSocket Hook（消息接收） |
| 操作日志 AOP | 无需前端实现（后端自动记录） |
