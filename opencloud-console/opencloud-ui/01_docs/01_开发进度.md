## 开发规范

- **组件命名**：PascalCase，文件名与组件名一致
- **API 函数**：每个模块一个文件，函数名 `get/create/update/delete` 前缀
- **TypeScript**：所有接口请求参数和响应类型需定义 `interface`，放在对应 `api/*.ts` 文件末尾
- **样式**：优先使用 Element Plus 变量，自定义样式写在 `<style scoped>` 中，全局样式只在 `index.css` 修改
- **权限控制**：使用 `v-permission` 指令或 `<PermissionButton>` 组件，不允许在页面逻辑中直接判断
- **动态路由**：登录后由 `user.ts` 的 `loadDynamicRoutes()` 从后端菜单接口加载；接口失败时自动降级为内置静态路由

## 本地开发

```bash
cd opencloud-ui
npm install
npm run dev         # 启动开发服务器（http://localhost:3000）
npm run build       # 构建生产包（dist/）
npm run preview     # 预览生产构建
```

Vite 开发服务器已配置代理：
- `/api/**` → `http://localhost:8080`（Spring Boot 后端）
- `/grafana/**` → `http://localhost:3000`（Grafana 实例）
