## 一、业务定位

`opencloud-ui` 是 OpenCloud 控制台的统一前端工程，为自建机房企业客户提供可视化、一站式的云计算基础设施管理界面。

主要面向系统管理员、运维工程师、开发人员等角色，提供：
- 统一的身份认证与权限控制
- 多云资源（OpenStack、Ceph、Kubernetes）的统一纳管
- 集中式的监控告警大盘
- 平台级的系统配置与审计

---

## 二、核心业务模块划分

### 2.1 认证与权限域
- **登录认证**：支持用户名/密码登录，基于 JWT 双 Token 机制。
- **动态路由**：登录后根据用户角色与权限，动态生成左侧菜单与可访问路由。
- **按钮级权限**：基于 `v-permission` 指令，控制页面内操作按钮的显隐。

### 2.2 仪表盘总览域
- **资源概览**：全局展示云主机数、节点数、存储容量、告警数等核心指标。
- **趋势分析**：通过 ECharts 展示 CPU、内存、网络等资源的 7 天使用趋势。
- **实时告警**：展示最新触发的监控告警信息。

### 2.3 云平台管理域
- **OpenStack 虚拟化**：
  - 云主机生命周期管理（创建、启停、重启、删除）。
  - 规格（Flavor）与镜像（Image）管理。
  - 网络（VPC、子网、安全组）与存储卷（Volume）管理。
- **Ceph 分布式存储**：
  - 存储集群状态与容量大盘。
  - 存储池（Pool）管理。
  - 对象存储（Bucket）管理。
- **Kubernetes 容器编排**：
  - 工作负载（Deployment、StatefulSet 等）查看。
  - Pod 状态监控与实时日志查看。
  - Service、ConfigMap、Node 等资源管理。

### 2.4 监控告警域
- **监控大盘**：集成 Grafana 面板，提供直观的指标可视化。
- **告警规则**：配置 Prometheus 告警规则（PromQL、阈值、持续时间）。
- **告警事件**：实时接收告警推送（WebSocket），支持告警确认（ACK）。
- **通知渠道**：配置邮件、钉钉、企业微信、Webhook 等通知方式。

### 2.5 系统管理域
- **用户管理**：平台用户的增删改查、密码重置、角色分配。
- **角色管理**：角色的定义及权限树（菜单/按钮）的分配。
- **菜单管理**：系统菜单树的维护，支持目录、菜单、按钮三种类型。
- **审计日志**：操作日志与登录日志的查询与追溯。

## 三、关键业务流程（前端视角）

### 3.1 登录与权限初始化流程
1. 用户在 `/login` 提交表单。
2. 调用 `/api/auth/login` 获取 `accessToken` 和 `refreshToken`，存入 `localStorage` 和 Pinia。
3. 获取用户信息及权限标识列表（`permissions`）。
4. 根据权限标识过滤路由表，动态添加路由（`router.addRoute`）。
5. 跳转至 `/dashboard`，渲染侧边栏菜单。

### 3.2 Token 无感刷新流程
1. Axios 响应拦截器捕获到 `401 Unauthorized` 错误。
2. 检查本地是否存在 `refreshToken`。
3. 挂起当前失败的请求，调用 `/api/auth/refresh` 接口换取新 Token。
4. 刷新成功后，更新本地 Token，并重试之前挂起的请求。
5. 刷新失败（如 Refresh Token 也过期），则清除状态跳转至登录页。

### 3.3 实时告警推送流程
1. 用户登录成功后，前端建立与 `/ws/alerts` 的 WebSocket 连接。
2. 连接建立时，拦截器自动校验 Header/Cookie 中的 JWT。
3. 前端定时（如 30s）发送 `PING` 心跳保活。
4. 接收到 `ALERT_FIRING` 消息时，触发全局 Notification 提示，并更新 Pinia 中的告警状态。
5. 接收到 `ALERT_RESOLVED` 消息时，更新状态并可自动关闭对应提示。

## 五、与总体业务设计的对应关系

### 5.1 前端在业务领域中的定位

根据总体业务设计文档 (`01_docs/02_总体业务设计.md`)，前端在各业务领域中的职责如下：

| 业务领域 | 后端模块 | 前端职责 | 关键交互 |
|---------|---------|---------|---------|
| **认证授权域** | opencloud-auth | 登录表单、Token 存储、路由守卫、权限控制 | 调用 `/api/auth/login` 获取 Token，401 时自动刷新 |
| **系统管理域** | opencloud-system | 用户/角色/菜单/部门的增删改查界面 | 表格CRUD、树形组件、权限树选择 |
| **云平台对接域** | opencloud-cloud | OpenStack/Ceph/K8s 资源的可视化与管理 | 列表展示、状态流转动画、异步任务进度 |
| **监控告警域** | opencloud-monitor | 指标查询、告警列表、渠道配置、WebSocket 实时推送 | Grafana 嵌入、PromQL 编辑器、告警通知铃铛 |

## 六、前端业务规则补充

### 6.1 通用业务规则

根据总体业务设计中的通用规则，前端需配合实现：

| 规则类型 | 前端实现方式 |
|---------|-------------|
| **多租户隔离** | 路由守卫校验 Token 中的 tenantId，请求头自动携带（通过 Axios 拦截器） |
| **逻辑删除** | 删除操作仅做前端隐藏，后端返回 200 后从列表中移除 |
| **数据权限** | 根据用户角色过滤可访问的数据范围（下拉选项动态过滤） |

### 6.2 安全相关规则

| 规则类型 | 前端实现方式 |
|---------|-------------|
| **Token 过期** | Axios 拦截器捕获 401，调用 Refresh Token 刷新 |
| **密码强度提示** | 密码输入框实时显示强度（弱/中/强） |
| **敏感操作确认** | 删除/重启等操作二次确认，防止误操作 |

### 6.3 云平台交互规则

| 规则类型 | 前端实现方式 |
|---------|-------------|
| **配额检查** | 创建资源前调用配额接口，超额则禁用提交按钮并提示 |
| **异步操作** | 创建后跳转到任务列表页，实时显示进度 |
| **状态同步** | 列表页支持手动刷新按钮，定时自动刷新（可选） |
