## 一、本地存储设计 (LocalStorage)

前端使用 `localStorage` 持久化关键状态，确保页面刷新后状态不丢失。

| Key | 类型 | 说明 | 生命周期 |
|-----|------|------|---------|
| `accessToken` | String | JWT 访问令牌 | 登录成功写入，登出/过期清除 |
| `refreshToken` | String | JWT 刷新令牌 | 登录成功写入，登出/过期清除 |
| `userInfo` | JSON String | 当前登录用户信息及权限 | 登录成功写入，登出清除 |
| `sidebarStatus` | String | 侧边栏折叠状态 ('opened'/'closed') | 长期有效 |
| `theme` | String | 主题偏好 ('light'/'dark') | 长期有效（预留） |

## 五、与总体数据设计的对应

### 5.1 前端数据流转架构

根据总体数据设计，前端在数据流转中的角色如下：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  前端表单    │────▶│  Axios 请求  │────▶│  后端 Controller │────▶│  数据库     │
│  用户输入    │     │  JSON 数据  │     │  业务处理     │     │  存储       │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │                   │
       ▼                   ▼                   ▼                   ▼
  本地校验          Token 附加          响应封装           前端渲染
```

### 5.2 后端数据表对应

前端展示的数据与后端数据库表的对应关系：

| 前端展示 | 后端 Entity | 数据库表 |
|---------|-------------|---------|
| 用户信息 | SysUser | sys_user |
| 角色信息 | SysRole | sys_role |
| 菜单树 | SysMenu | sys_menu |
| 云主机列表 | InstanceVO | nova servers (OpenStack) |
| 存储池 | PoolVO | ceph pools (Ceph) |
| 告警记录 | AlertRecord | monitor_alert_record |
| 通知渠道 | NotifyChannel | monitor_notify_channel |

### 5.3 分页查询流程

前端发起分页请求到后端数据表的完整流程：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  前端分页    │     │  MyBatis    │     │  PostgreSQL │     │  返回响应   │
│  page=1     │────▶│  Plus 分页   │────▶│  LIMIT 10   │────▶│  list+total │
│  size=10    │     │  拦截器     │     │  OFFSET 0   │     │  前端渲染   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

### 5.4 多租户数据隔离

前端需配合后端实现多租户数据隔离：

1. **Token 中包含 tenantId**：登录成功后，JWT 中包含 `tenantId` claim。
2. **请求自动携带**：Axios 拦截器将 tenantId 附加到请求头 `X-Tenant-ID`。
3. **后端拦截**：MyBatis-Plus 租户拦截器自动在 SQL 中追加 `tenant_id = ?` 条件。
4. **前端无感知**：用户无感知，数据自动按租户隔离。

## 六、数据处理最佳实践

### 6.1 数据缓存策略

前端可利用 `localStorage` 和 `sessionStorage` 缓存部分数据：

| 数据类型 | 存储方式 | TTL | 说明 |
|---------|---------|-----|------|
| Token | localStorage | 2 小时 | Access Token |
| 刷新 Token | localStorage | 7 天 | 用于刷新 |
| 用户信息 | sessionStorage | 页面关闭 | 敏感信息 |
| 字典数据 | memory (Pinia) | 1 小时 | 可通过后端接口刷新 |
| 菜单树 | memory (Pinia) | 持久 | 登录后加载 |

### 6.2 大数据量处理

对于云主机日志、监控指标等大数据量场景：

1. **分页加载**：严格使用分页，避免一次性加载全部数据。
2. **虚拟滚动**：长列表场景使用 `vue-virtual-scroller` 虚拟滚动。
3. **增量请求**：PromQL 查询支持 `start/end/step` 增量拉取。
4. **WebSocket 流**：日志、实时告警使用 WebSocket 推送，避免轮询。

### 6.3 数据脱敏展示

后端返回的敏感数据，前端需脱敏展示：

| 字段 | 原始值 | 脱敏展示 |
|-----|--------|---------|
| 手机号 | 13812345678 | 138****5678 |
| 邮箱 | user@example.com | u***@example.com |
| 密钥 | sk-xxx-xxx | ********** |
