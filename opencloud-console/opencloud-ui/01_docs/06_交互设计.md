# opencloud-ui — 交互设计

> 前端页面布局、色彩规范与核心交互逻辑
> 
> 版本：v1.0 | 最后更新：2026-03-31

---

## 一、视觉与主题规范

### 1.1 设计语言

OpenCloud 控制台采用**"科技、专业、清晰"**的设计语言，整体风格偏向现代化企业级 B 端产品，类似阿里云/腾讯云控制台。

### 1.2 色彩规范

基于 Element Plus 的色彩体系进行定制：

- **主色调 (Primary)**：`#409EFF` (科技蓝) - 用于主要按钮、激活状态、链接。
- **成功色 (Success)**：`#67C23A` (绿色) - 用于运行中、成功状态。
- **警告色 (Warning)**：`#E6A23C` (橙色) - 用于创建中、警告告警。
- **危险色 (Danger)**：`#F56C6C` (红色) - 用于异常、错误、严重告警、删除操作。
- **信息色 (Info)**：`#909399` (灰色) - 用于离线、禁用状态、次要文本。
- **背景色**：`#F5F7FA` (浅灰) - 用于页面大背景，突出白色卡片内容。

### 1.3 字体规范

- **字体家族**：`'Helvetica Neue', Helvetica, 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', '微软雅黑', Arial, sans-serif`
- **字号**：
  - 标题：`20px` / `18px` / `16px`
  - 正文：`14px` (默认)
  - 辅助文本：`12px`

## 二、整体布局设计 (MainLayout)

采用经典的**"左侧导航 + 顶部信息 + 右侧内容区"**布局。

### 2.1 侧边栏 (Sidebar)
- **宽度**：展开时 `220px`，折叠时 `64px`。
- **交互**：
  - 顶部显示 Logo 和系统名称。
  - 支持点击折叠/展开按钮（位于顶栏）。
  - 菜单项支持多级嵌套，当前激活项高亮显示并自动展开父级。
  - 采用深色背景（`#304156`）以区分内容区。

### 2.2 顶栏 (Header)
- **高度**：`50px`。
- **左侧**：折叠/展开侧边栏的汉堡按钮，以及动态面包屑导航（Breadcrumb）。
- **右侧**：
  - 告警通知铃铛（带未读红点徽标）。
  - 用户头像与下拉菜单（包含"个人中心"、"退出登录"）。

### 2.3 内容区 (AppMain)
- **背景**：浅灰色背景，内容区域使用白色卡片（`el-card`）包裹。
- **边距**：四周统一 `20px` 内边距。
- **过渡动画**：路由切换时使用 `fade-transform` 渐变动画。

## 三、核心页面交互设计

### 3.1 登录页 (Login)
- **视觉**：全屏暗色科技感背景（或高质量抽象背景图），中央放置毛玻璃效果（Glassmorphism）的登录卡片。
- **交互**：
  - 输入框支持回车键（Enter）提交。
  - 提交时按钮显示 Loading 状态。
  - 错误提示使用 `ElMessage` 在顶部居中弹出。

### 3.2 仪表盘 (Dashboard)
- **顶部**：4 个核心指标卡片（MetricCard），展示数字及小趋势图。
- **中部**：宽幅折线图，展示集群整体资源（CPU/内存）趋势。
- **底部**：分栏布局，左侧为存储分布饼图，右侧为最新告警列表。
- **交互**：图表支持鼠标悬浮（Tooltip）查看具体数值。

### 3.3 列表管理页 (如云主机列表)
采用标准 CRUD 页面布局：
- **顶部搜索区**：内联表单（Inline Form），包含关键字输入、状态下拉框及"查询/重置"按钮。
- **操作按钮区**：位于表格左上方，包含"新建"、"批量删除"等按钮。
- **数据表格**：
  - 状态列使用 `StatusBadge`（带颜色的小圆点+文字）展示。
  - 操作列固定在最右侧，包含"详情"、"编辑"、"删除"等文字按钮（Link Button）。
- **底部分页**：右下角对齐，展示总条数，支持切换每页条数。

### 3.4 表单弹窗 (Dialog)
- **交互**：
  - 新建/编辑操作统一使用 `el-dialog` 弹窗处理，避免页面跳转打断上下文。
  - 弹窗宽度适中（通常 `500px` - `700px`）。
  - 底部固定"取消"和"确定"按钮。
  - 点击"确定"时触发整表校验，校验不通过则标红提示，通过则按钮进入 Loading 状态。

## 五、与总体业务设计的交互对应

### 5.1 业务流程交互映射

前端交互设计需严格遵循总体业务设计文档 (`01_docs/02_总体业务设计.md`) 中的业务流程：

#### 用户认证流程交互

根据总体设计中"2.2.1 用户认证流程"，前端交互实现：

```
输入账号密码 → 提交 POST /api/auth/login 
  → 200 返回 {accessToken, refreshToken} 
  → 存储 Token 到 localStorage 
  → 获取用户信息 /api/auth/userinfo 
  → 解析权限 permissions[] 
  → 动态生成路由 
  → 跳转 /dashboard
```

**错误处理：**
- 密码错误 5 次 → 后端返回 423 锁定，前端提示"账号已锁定，请 30 分钟后重试"
- 密码错误 < 5 次 → 提示"用户名或密码错误，剩余 X 次"

#### 云主机创建流程交互

根据总体设计中"2.2.2 资源管理流程（以云主机为例）"，前端交互实现：

```
点击"创建云主机" → 打开步骤向导 Dialog
  → 第1步：选择规格（flavor）/镜像（image）
  → 第2步：选择网络（network）/安全组（security groups）
  → 第3步：配置登录方式（密钥对/密码）
  → 点击"创建" → POST /api/cloud/openstack/instances
  → 返回 {taskId} → 跳转到云主机列表
  → 轮询任务状态 或 WebSocket 接收进度
  → 状态变为 ACTIVE → 提示"创建成功"
```

#### 告警处理流程交互

根据总体设计中"2.2.3 告警处理流程"，前端交互实现：

```
WebSocket 连接 /ws/alerts
  → 接收 ALERT_FIRING 消息
  → 触发全局 Notification 提示（根据 severity 显示不同颜色）
  → 铃铛图标未读数 +1
  → 点击铃铛打开告警列表 Drawer
  → 用户点击"确认" → POST /api/monitor/alert/records/{id}/ack
  → 状态变为 ACKNOWLEDGED
```

## 六、用户体验细节

### 6.1 加载状态设计

| 场景 | 状态 | 视觉实现 |
|------|------|----------|
| 页面首次加载 | 骨架屏 | `el-skeleton` 组件 |
| 表格数据加载 | 表格 loading | `v-loading` 指令 |
| 按钮提交中 | 按钮 loading | `el-button` 的 `loading` 属性 |
| 云资源操作 | 进度条 | `el-progress` + 状态文字 |
| 页面跳转 | 顶部进度条 | NProgress |

### 6.2 空状态设计

各业务模块的空状态展示：

| 模块 | 空状态文案 | 建议操作 |
|------|-----------|---------|
| 用户管理 | 暂无用户 | 点击"新增用户" |
| 云主机列表 | 暂无云主机 | 点击"创建云主机" |
| 告警记录 | 暂无告警 | — |
| 操作日志 | 暂无日志 | — |

### 6.3 表格设计规范

- **行高**：固定 52px，保证视觉一致性
- **斑马纹**：可选开启，默认关闭以提升性能
- **单元格省略**：超长文本显示省略号，Hover 显示 Tooltip
- **操作列**：固定在最后一列，宽度 180px，包含 2-3 个操作

### 6.4 表单设计规范

- **标签对齐**：右对齐，宽度 100px
- **输入框宽度**：默认 300px，长文本使用 500px
- **必填标识**：标签前加红色 `*` 星号
- **校验提示**：底部红色文字，紧凑布局
- **按钮组**：底部居右，"取消" + "确定"

## 七、可访问性设计

### 7.1 键盘导航

- Tab 键可遍历所有可交互元素
- Enter 键触发按钮点击/表单提交
- Escape 键关闭 Dialog/Drawer
- 方向键在 Select 下拉/Tree 节点中导航

### 7.2 屏幕阅读器

- 所有图片添加 `alt` 属性
- 按钮图标添加 `aria-label`
- 表格添加表头 `scope="col"`
- 状态标签添加语义化角色

### 7.3 颜色对比度

- 文字与背景对比度 ≥ 4.5:1
- 聚焦状态使用明显的 outline 或 border

## 八、响应式布局

### 8.1 断点设计

| 断点 | 宽度 | 布局变化 |
|------|------|----------|
| Mobile | < 768px | 侧边栏隐藏，顶部 hamburger 触发 |
| Tablet | 768px - 1200px | 侧边栏默认折叠 |
| Desktop | > 1200px | 侧边栏默认展开 |

### 8.2 适配策略

- 使用 Element Plus 的 `el-col` 响应式栅格
- 关键交互元素（按钮/表单）避免放置在最右侧
- 表格在移动端自动转为卡片式布局
