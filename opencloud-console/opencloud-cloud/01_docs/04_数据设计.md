# opencloud-cloud 数据设计文档

---

## 1. 数据库选型

| 存储 | 用途 |
|------|------|
| PostgreSQL | 集群配置、异步任务、配额记录等持久化数据 |
| Redis | 客户端缓存、配额缓存、任务状态、同步锁 |

Schema 名：`cloud`

---

## 2. 数据表设计

### 2.1 集群配置表 `cloud_cluster_config`

> 存储 OpenStack、Ceph、Kubernetes 的连接配置信息，敏感字段加密存储。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | bigserial | PK | 自增主键 |
| tenant_id | varchar(36) | NOT NULL | 租户 ID |
| name | varchar(64) | NOT NULL | 集群名称（租户内唯一） |
| type | varchar(16) | NOT NULL | 集群类型：OPENSTACK / CEPH / KUBERNETES |
| endpoint | varchar(256) | NOT NULL | 服务端点 URL |
| config_json | text | NOT NULL | 连接配置 JSON（AES 加密） |
| status | varchar(16) | NOT NULL, DEFAULT 'PENDING' | 状态：ACTIVE / ERROR / PENDING |
| last_check_time | timestamptz | NULL | 上次连接测试时间 |
| error_msg | varchar(512) | NULL | 错误信息（连接失败时记录） |
| created_at | timestamptz | NOT NULL, DEFAULT now() | 创建时间 |
| updated_at | timestamptz | NOT NULL, DEFAULT now() | 更新时间 |
| created_by | varchar(36) | NOT NULL | 创建人 ID |
| updated_by | varchar(36) | NOT NULL | 更新人 ID |
| deleted | smallint | NOT NULL, DEFAULT 0 | 逻辑删除标记：0=未删除 1=已删除 |

**字段说明：**
- `config_json` 存储加密后的 JSON 字符串，包含各平台特定的连接参数
- `status` 在连接测试成功后更新为 ACTIVE，失败后更新为 ERROR
- `error_msg` 在连接失败时记录具体错误信息，便于排查

**索引：**
```sql
CREATE UNIQUE INDEX idx_ccc_name ON cloud_cluster_config(tenant_id, name) WHERE deleted = 0;
CREATE INDEX idx_ccc_tenant ON cloud_cluster_config(tenant_id);
CREATE INDEX idx_ccc_type ON cloud_cluster_config(type);
CREATE INDEX idx_ccc_status ON cloud_cluster_config(status);
```

### 2.2 异步任务表 `cloud_task`

> 记录云资源异步操作的任务状态，用于跟踪创建、删除等耗时操作。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | bigserial | PK | 自增主键 |
| task_id | varchar(36) | NOT NULL, UNIQUE | 任务唯一标识（UUID） |
| tenant_id | varchar(36) | NOT NULL | 租户 ID |
| type | varchar(32) | NOT NULL | 任务类型：INSTANCE_CREATE / INSTANCE_DELETE / VOLUME_CREATE / VOLUME_DELETE / IMAGE_UPLOAD / ... |
| status | varchar(16) | NOT NULL, DEFAULT 'PENDING' | 状态：PENDING / RUNNING / SUCCESS / FAILED / CANCELLED |
| resource_type | varchar(32) | NOT NULL | 资源类型：INSTANCE / VOLUME / IMAGE / ... |
| resource_id | varchar(64) | NULL | 资源 ID（云平台返回的 ID） |
| cluster_id | varchar(36) | NOT NULL | 关联集群 ID |
| progress | smallint | NOT NULL, DEFAULT 0 | 进度百分比（0-100） |
| result_json | text | NULL | 执行结果 JSON（成功时） |
| error_msg | varchar(512) | NULL | 错误信息（失败时） |
| created_at | timestamptz | NOT NULL, DEFAULT now() | 创建时间 |
| started_at | timestamptz | NULL | 开始执行时间 |
| completed_at | timestamptz | NULL | 完成时间 |
| created_by | varchar(36) | NOT NULL | 创建人 ID |

**任务类型枚举：**

| 值 | 说明 |
|----|------|
| `INSTANCE_CREATE` | 创建云主机 |
| `INSTANCE_DELETE` | 删除云主机 |
| `INSTANCE_RESIZE` | 调整云主机规格 |
| `VOLUME_CREATE` | 创建卷 |
| `VOLUME_DELETE` | 删除卷 |
| `VOLUME_ATTACH` | 挂载卷 |
| `VOLUME_DETACH` | 卸载卷 |
| `IMAGE_UPLOAD` | 上传镜像 |
| `IMAGE_DELETE` | 删除镜像 |
| `CLUSTER_SYNC` | 集群资源同步 |

**索引：**
```sql
CREATE UNIQUE INDEX idx_ct_task_id ON cloud_task(task_id);
CREATE INDEX idx_ct_tenant ON cloud_task(tenant_id);
CREATE INDEX idx_ct_status ON cloud_task(status);
CREATE INDEX idx_ct_type ON cloud_task(type);
CREATE INDEX idx_ct_cluster ON cloud_task(cluster_id);
CREATE INDEX idx_ct_resource ON cloud_task(resource_type, resource_id);
CREATE INDEX idx_ct_created ON cloud_task(created_at DESC);
```

**数据保留策略：** 已完成任务保留 7 天，超期自动清理。

### 2.3 配额配置表 `cloud_quota`

> 存储租户在各平台的资源配额限制。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | bigserial | PK | 自增主键 |
| tenant_id | varchar(36) | NOT NULL, UNIQUE | 租户 ID |
| openstack_vcpu_limit | integer | NOT NULL, DEFAULT -1 | OpenStack vCPU 限制（-1=无限制） |
| openstack_memory_limit | bigint | NOT NULL, DEFAULT -1 | OpenStack 内存限制（MB，-1=无限制） |
| openstack_instance_limit | integer | NOT NULL, DEFAULT -1 | OpenStack 实例数限制（-1=无限制） |
| openstack_volume_count_limit | integer | NOT NULL, DEFAULT -1 | OpenStack 卷数量限制（-1=无限制） |
| openstack_volume_storage_limit | bigint | NOT NULL, DEFAULT -1 | OpenStack 卷存储限制（GB，-1=无限制） |
| ceph_storage_limit | bigint | NOT NULL, DEFAULT -1 | Ceph 存储限制（GB，-1=无限制） |
| k8s_cpu_limit | integer | NOT NULL, DEFAULT -1 | K8s CPU 限制（核，-1=无限制） |
| k8s_memory_limit | bigint | NOT NULL, DEFAULT -1 | K8s 内存限制（MB，-1=无限制） |
| created_at | timestamptz | NOT NULL, DEFAULT now() | 创建时间 |
| updated_at | timestamptz | NOT NULL, DEFAULT now() | 更新时间 |

**字段说明：**
- 所有 limit 字段默认值为 -1，表示无限制
- 新租户默认配额为 0（需管理员手动分配）

**索引：**
```sql
CREATE UNIQUE INDEX idx_cq_tenant ON cloud_quota(tenant_id);
```

### 2.4 资源同步日志表 `cloud_sync_log`

> 记录集群资源同步的历史记录，用于审计和排查同步问题。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | bigserial | PK | 自增主键 |
| cluster_id | varchar(36) | NOT NULL | 集群 ID |
| tenant_id | varchar(36) | NOT NULL | 租户 ID |
| sync_type | varchar(16) | NOT NULL | 同步类型：AUTO / MANUAL |
| status | varchar(16) | NOT NULL | 状态：SUCCESS / FAILED |
| resource_counts | jsonb | NULL | 各资源类型同步数量统计 |
| error_msg | varchar(512) | NULL | 错误信息（失败时） |
| started_at | timestamptz | NOT NULL | 同步开始时间 |
| completed_at | timestamptz | NULL | 同步完成时间 |
| created_at | timestamptz | NOT NULL, DEFAULT now() | 记录创建时间 |

**resource_counts JSON 示例：**
```json
{
  "instances": 50,
  "volumes": 30,
  "networks": 10,
  "images": 20
}
```

**索引：**
```sql
CREATE INDEX idx_csl_cluster ON cloud_sync_log(cluster_id);
CREATE INDEX idx_csl_tenant ON cloud_sync_log(tenant_id);
CREATE INDEX idx_csl_status ON cloud_sync_log(status);
CREATE INDEX idx_csl_created ON cloud_sync_log(created_at DESC);
```

**数据保留策略：** 保留最近 30 天数据。

## 3. Redis 数据结构说明

> 所有 Key 以 `cloud:` 为统一命名空间前缀。

| Key 模式 | 类型 | 示例值 | TTL | 说明 |
|----------|------|--------|-----|------|
| `cloud:client:{clusterId}` | String | 序列化后的客户端对象 | 永久 | 云平台客户端实例缓存，配置变更时清除 |
| `cloud:quota:{tenantId}` | String | JSON 序列化的配额对象 | 5 分钟 | 租户配额缓存 |
| `cloud:usage:{tenantId}:{clusterId}` | String | JSON 序列化的使用量 | 5 分钟 | 资源使用量缓存 |
| `cloud:task:{taskId}` | String | JSON 序列化的任务状态 | 任务完成后 7 天 | 任务状态缓存 |
| `cloud:sync:{clusterId}` | String | "1" | 10 分钟 | 同步锁，防止并发同步同一集群 |
| `cloud:sync:last:{clusterId}` | String | 时间戳 | 永久 | 上次同步时间 |

**Redis Key 操作规范：**

| 场景 | Redis 操作 |
|------|-----------|
| 获取客户端 | `GET cloud:client:{clusterId}` → 未命中则创建并 `SET` |
| 配置变更 | `DEL cloud:client:{clusterId}` |
| 获取配额 | `GET cloud:quota:{tenantId}` → 未命中则查 DB 并 `SETEX 300` |
| 更新配额 | 更新 DB 后 `DEL cloud:quota:{tenantId}` |
| 获取使用量 | `GET cloud:usage:{tenantId}:{clusterId}` → 未命中则统计并 `SETEX 300` |
| 创建任务 | 插入 DB 后 `SETEX cloud:task:{taskId} 604800 {taskJson}` |
| 更新任务 | 更新 DB 后 `SETEX cloud:task:{taskId} 604800 {taskJson}` |
| 同步开始 | `SETEX cloud:sync:{clusterId} 600 "1"` |
| 同步结束 | `DEL cloud:sync:{clusterId}`；`SET cloud:sync:last:{clusterId} {timestamp}` |

## 4. 数据流向说明

```
新增集群配置（POST /api/cloud/clusters）
  ↓
参数校验 → 敏感信息 AES 加密
  ↓
写入 cloud_cluster_config
  ↓
写入操作日志

──────────────────────────────────────────

连接测试（POST /api/cloud/clusters/{id}/test）
  ↓
读取 cloud_cluster_config → 解密 config_json
  ↓
创建客户端并测试连接
  ↓
更新 cloud_cluster_config.status / last_check_time / error_msg
  ↓
如成功：缓存客户端到 Redis cloud:client:{id}

──────────────────────────────────────────

创建云主机（POST /api/cloud/openstack/instances）
  ↓
配额检查
  ├─ 读 Redis cloud:quota:{tenantId}（未命中读 DB）
  ├─ 读 Redis cloud:usage:{tenantId}:{clusterId}（未命中统计）
  └─ 校验配额
  ↓
创建异步任务 → 写入 cloud_task
  ↓
异步执行
  ├─ 更新 cloud_task.status = RUNNING
  ├─ 调用 OpenStack API 创建云主机
  ├─ 轮询状态并更新 cloud_task.progress
  └─ 完成时更新 cloud_task.status = SUCCESS / FAILED
  ↓
清除使用量缓存 DEL cloud:usage:{tenantId}:{clusterId}

──────────────────────────────────────────

定时资源同步（ResourceSyncJob）
  ↓
获取所有 ACTIVE 状态的 cloud_cluster_config
  ↓
对每个集群
  ├─ 尝试获取锁 SETEX cloud:sync:{clusterId} 600 "1"（失败则跳过）
  ├─ 同步各类资源
  ├─ 写入 cloud_sync_log
  ├─ 释放锁 DEL cloud:sync:{clusterId}
  └─ 更新最后同步时间 SET cloud:sync:last:{clusterId} {timestamp}
  ↓
清除相关使用量缓存

──────────────────────────────────────────

获取配额（GET /api/cloud/quotas/{tenantId}）
  ↓
读 Redis cloud:quota:{tenantId}（命中直接返回）
  ↓
未命中 → 读 cloud_quota → 写入 Redis SETEX 300
  ↓
统计当前使用量
  ├─ 读 Redis cloud:usage:{tenantId}:{clusterId}（命中直接返回）
  ├─ 未命中 → 查询云平台统计 → 写入 Redis SETEX 300
  ↓
返回配额详情
```

## 5. 数据初始化脚本（DDL）

```sql
-- ============================================================
-- Schema
-- ============================================================
CREATE SCHEMA IF NOT EXISTS cloud;

-- ============================================================
-- 集群配置表
-- ============================================================
CREATE TABLE cloud.cloud_cluster_config (
    id                  bigserial       PRIMARY KEY,
    tenant_id           varchar(36)     NOT NULL,
    name                varchar(64)     NOT NULL,
    type                varchar(16)     NOT NULL,  -- OPENSTACK / CEPH / KUBERNETES
    endpoint            varchar(256)    NOT NULL,
    config_json         text            NOT NULL,  -- AES 加密的 JSON
    status              varchar(16)     NOT NULL DEFAULT 'PENDING',  -- ACTIVE / ERROR / PENDING
    last_check_time     timestamptz,
    error_msg           varchar(512),
    created_at          timestamptz     NOT NULL DEFAULT now(),
    updated_at          timestamptz     NOT NULL DEFAULT now(),
    created_by          varchar(36)     NOT NULL,
    updated_by          varchar(36)     NOT NULL,
    deleted             smallint        NOT NULL DEFAULT 0
);

CREATE UNIQUE INDEX idx_ccc_name    ON cloud.cloud_cluster_config(tenant_id, name) WHERE deleted = 0;
CREATE INDEX        idx_ccc_tenant  ON cloud.cloud_cluster_config(tenant_id);
CREATE INDEX        idx_ccc_type    ON cloud.cloud_cluster_config(type);
CREATE INDEX        idx_ccc_status  ON cloud.cloud_cluster_config(status);

COMMENT ON TABLE  cloud.cloud_cluster_config           IS '云平台集群配置表';
COMMENT ON COLUMN cloud.cloud_cluster_config.type      IS '集群类型: OPENSTACK / CEPH / KUBERNETES';
COMMENT ON COLUMN cloud.cloud_cluster_config.status    IS '状态: ACTIVE=正常 ERROR=连接失败 PENDING=待测试';
COMMENT ON COLUMN cloud.cloud_cluster_config.config_json IS '连接配置 JSON（AES-256-GCM 加密）';

-- ============================================================
-- 异步任务表
-- ============================================================
CREATE TABLE cloud.cloud_task (
    id              bigserial       PRIMARY KEY,
    task_id         varchar(36)     NOT NULL UNIQUE,
    tenant_id       varchar(36)     NOT NULL,
    type            varchar(32)     NOT NULL,  -- INSTANCE_CREATE / VOLUME_CREATE / ...
    status          varchar(16)     NOT NULL DEFAULT 'PENDING',  -- PENDING / RUNNING / SUCCESS / FAILED / CANCELLED
    resource_type   varchar(32)     NOT NULL,
    resource_id     varchar(64),
    cluster_id      varchar(36)     NOT NULL,
    progress        smallint        NOT NULL DEFAULT 0,
    result_json     text,
    error_msg       varchar(512),
    created_at      timestamptz     NOT NULL DEFAULT now(),
    started_at      timestamptz,
    completed_at    timestamptz,
    created_by      varchar(36)     NOT NULL
);

CREATE UNIQUE INDEX idx_ct_task_id   ON cloud.cloud_task(task_id);
CREATE INDEX        idx_ct_tenant    ON cloud.cloud_task(tenant_id);
CREATE INDEX        idx_ct_status    ON cloud.cloud_task(status);
CREATE INDEX        idx_ct_type      ON cloud.cloud_task(type);
CREATE INDEX        idx_ct_cluster   ON cloud.cloud_task(cluster_id);
CREATE INDEX        idx_ct_resource  ON cloud.cloud_task(resource_type, resource_id);
CREATE INDEX        idx_ct_created   ON cloud.cloud_task(created_at DESC);

COMMENT ON TABLE  cloud.cloud_task              IS '异步任务表，跟踪云资源操作';
COMMENT ON COLUMN cloud.cloud_task.type         IS '任务类型: INSTANCE_CREATE / INSTANCE_DELETE / VOLUME_CREATE / VOLUME_DELETE / IMAGE_UPLOAD / CLUSTER_SYNC / ...';
COMMENT ON COLUMN cloud.cloud_task.status       IS '状态: PENDING / RUNNING / SUCCESS / FAILED / CANCELLED';
COMMENT ON COLUMN cloud.cloud_task.progress     IS '进度百分比（0-100）';

-- ============================================================
-- 配额配置表
-- ============================================================
CREATE TABLE cloud.cloud_quota (
    id                              bigserial       PRIMARY KEY,
    tenant_id                       varchar(36)     NOT NULL UNIQUE,
    openstack_vcpu_limit            integer         NOT NULL DEFAULT -1,
    openstack_memory_limit          bigint          NOT NULL DEFAULT -1,
    openstack_instance_limit        integer         NOT NULL DEFAULT -1,
    openstack_volume_count_limit    integer         NOT NULL DEFAULT -1,
    openstack_volume_storage_limit  bigint          NOT NULL DEFAULT -1,
    ceph_storage_limit              bigint          NOT NULL DEFAULT -1,
    k8s_cpu_limit                   integer         NOT NULL DEFAULT -1,
    k8s_memory_limit                bigint          NOT NULL DEFAULT -1,
    created_at                      timestamptz     NOT NULL DEFAULT now(),
    updated_at                      timestamptz     NOT NULL DEFAULT now()
);

CREATE UNIQUE INDEX idx_cq_tenant ON cloud.cloud_quota(tenant_id);

COMMENT ON TABLE  cloud.cloud_quota                           IS '租户资源配额表';
COMMENT ON COLUMN cloud.cloud_quota.openstack_vcpu_limit      IS 'OpenStack vCPU 限制（-1=无限制）';
COMMENT ON COLUMN cloud.cloud_quota.openstack_memory_limit    IS 'OpenStack 内存限制（MB，-1=无限制）';
COMMENT ON COLUMN cloud.cloud_quota.ceph_storage_limit        IS 'Ceph 存储限制（GB，-1=无限制）';
COMMENT ON COLUMN cloud.cloud_quota.k8s_cpu_limit             IS 'K8s CPU 限制（核，-1=无限制）';

-- ============================================================
-- 资源同步日志表
-- ============================================================
CREATE TABLE cloud.cloud_sync_log (
    id                bigserial       PRIMARY KEY,
    cluster_id        varchar(36)     NOT NULL,
    tenant_id         varchar(36)     NOT NULL,
    sync_type         varchar(16)     NOT NULL,  -- AUTO / MANUAL
    status            varchar(16)     NOT NULL,  -- SUCCESS / FAILED
    resource_counts   jsonb,
    error_msg         varchar(512),
    started_at        timestamptz     NOT NULL,
    completed_at      timestamptz,
    created_at        timestamptz     NOT NULL DEFAULT now()
);

CREATE INDEX idx_csl_cluster  ON cloud.cloud_sync_log(cluster_id);
CREATE INDEX idx_csl_tenant   ON cloud.cloud_sync_log(tenant_id);
CREATE INDEX idx_csl_status   ON cloud.cloud_sync_log(status);
CREATE INDEX idx_csl_created  ON cloud.cloud_sync_log(created_at DESC);

COMMENT ON TABLE  cloud.cloud_sync_log            IS '资源同步日志表';
COMMENT ON COLUMN cloud.cloud_sync_log.sync_type  IS '同步类型: AUTO=自动定时同步 MANUAL=手动触发';
COMMENT ON COLUMN cloud.cloud_sync_log.resource_counts IS '各资源类型同步数量统计 JSON';
```

## 6. 表与业务流程对照

| 业务流程 | 读表 | 写表 | Redis 操作 |
|---------|------|------|-----------|
| 新增集群配置 | — | `cloud_cluster_config` | — |
| 编辑集群配置 | `cloud_cluster_config` | `cloud_cluster_config` | DEL client |
| 删除集群配置 | `cloud_cluster_config` | `cloud_cluster_config`（逻辑删除） | DEL client |
| 连接测试 | `cloud_cluster_config` | `cloud_cluster_config`（更新 status） | SET client |
| 创建云主机 | `cloud_cluster_config`、`cloud_quota` | `cloud_task` | GET/SET quota、GET/SET usage、SET task |
| 查询任务 | `cloud_task` | — | GET task |
| 更新任务 | `cloud_task` | `cloud_task` | SET task |
| 获取配额 | `cloud_quota` | — | GET/SET quota |
| 设置配额 | `cloud_quota` | `cloud_quota` | DEL quota |
| 定时同步 | `cloud_cluster_config` | `cloud_sync_log` | SET sync 锁、SET last 时间、DEL usage |
| 手动同步 | `cloud_cluster_config` | `cloud_sync_log` | SET sync 锁、SET last 时间 |
