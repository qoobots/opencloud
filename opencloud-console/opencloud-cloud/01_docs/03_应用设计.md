# opencloud-cloud 应用设计文档

---

## 1. 模块定位

`opencloud-cloud` 是 OpenCloud 云管平台的**云平台对接适配层**，职责边界：

- **对外**：提供 OpenStack、Ceph、Kubernetes 三大平台的资源管理 REST 接口
- **对内**：维护集群连接配置（`cluster_config`）、异步任务（`cloud_task`）、配额记录（`cloud_quota`）
- **依赖**：
  - PostgreSQL（持久化集群配置、任务、配额）
  - Redis（客户端缓存、配额缓存、任务状态）
  - opencloud-auth（Token 鉴权、权限校验）
  - opencloud-common（统一响应体、加密工具、异常处理）

## 3. 对外 API 设计

> **路径规范**：所有接口统一以 `/api/cloud` 为前缀。

```

  "code": 0,
  "data": {
    "total": 10,
    "pages": 1,
    "current": 1,
    "records": [
      {
        "id": "c001",
        "name": "生产 OpenStack",
        "type": "OPENSTACK",
        "endpoint": "http://192.168.1.10:5000/v3",

        "createdAt": "2026-01-15T10:00:00Z"
```

## 5. 安全配置要点

### 5.1 接口权限要求

| 接口 | 所需权限 |
|------|---------|
| `GET /api/cloud/clusters` | cloud:cluster:list |
| `POST /api/cloud/clusters` | cloud:cluster:add |
| `PUT /api/cloud/clusters/{id}` | cloud:cluster:edit |
| `DELETE /api/cloud/clusters/{id}` | cloud:cluster:delete |
| `GET /api/cloud/openstack/instances` | cloud:instance:list |
| `POST /api/cloud/openstack/instances` | cloud:instance:create |
| `PUT /api/cloud/openstack/instances/{id}/action` | cloud:instance:operate |
| `DELETE /api/cloud/openstack/instances/{id}` | cloud:instance:delete |
| `GET /api/cloud/quotas/{tenantId}` | cloud:quota:view |
| `PUT /api/cloud/quotas/{tenantId}` | cloud:quota:edit |

### 5.2 数据安全

- **敏感信息加密**：集群配置中的密码、密钥、Token 使用 AES-256-GCM 加密存储
- **加密密钥管理**：通过配置中心注入 `opencloud.encrypt.key`，不硬编码
- **脱敏展示**：API 返回时敏感字段脱敏显示（如 `password: "***"`）
- **客户端缓存**：云平台客户端按 clusterId 缓存，避免重复创建连接

### 5.3 访问控制

- 所有接口需携带有效 AccessToken
- 超级管理员可操作所有租户数据
- 租户管理员仅能操作本租户数据
- 数据查询自动附加 `tenant_id` 条件

## 6. Redis Key 规范

| Key 模式 | 说明 | TTL |
|----------|------|-----|
| `cloud:client:{clusterId}` | 云平台客户端实例缓存 | 永久（配置变更时清除） |
| `cloud:quota:{tenantId}` | 租户配额缓存 | 5 分钟 |
| `cloud:usage:{tenantId}:{clusterId}` | 资源使用量缓存 | 5 分钟 |
| `cloud:task:{taskId}` | 任务状态缓存 | 任务完成后 7 天 |
| `cloud:sync:{clusterId}` | 同步锁（防止并发同步） | 10 分钟 |

## 7. 异常码规范

> 格式统一为 `CLOUD_XXXX` 或 `OS_XXXX` / `CEPH_XXXX` / `K8S_XXXX` / `TASK_XXXX`

### 7.1 通用异常（CLOUD_xxxx）

| 错误码 | HTTP 状态 | 含义 |
|--------|-----------|------|
| `CLOUD_0001` | 404 | 集群配置不存在 |
| `CLOUD_0002` | 503 | 集群连接超时 |
| `CLOUD_0003` | 401 | 集群认证失败 |
| `CLOUD_0004` | 403 | 无权访问该集群 |
| `CLOUD_0005` | 503 | 集群当前离线 |
| `CLOUD_0101` | 404 | 资源不存在 |
| `CLOUD_0102` | 403 | 无权操作该资源 |
| `CLOUD_0103` | 400 | 当前状态不允许该操作 |
| `CLOUD_0104` | 400 | 配额不足 |
| `CLOUD_0105` | 409 | 资源名称冲突 |
| `CLOUD_0106` | 502 | 云平台接口调用失败 |

### 7.2 OpenStack 异常（OS_xxxx）

| 错误码 | HTTP 状态 | 含义 |
|--------|-----------|------|
| `OS_0001` | 400 | 规格不可用或不存在 |
| `OS_0002` | 400 | 镜像不可用或不存在 |
| `OS_0003` | 400 | 网络不可用或不存在 |
| `OS_0004` | 409 | 安全组规则冲突 |
| `OS_0005` | 400 | 浮动 IP 池已耗尽 |
| `OS_0006` | 409 | 卷已挂载到其他实例 |

### 7.3 Ceph 异常（CEPH_xxxx）

| 错误码 | HTTP 状态 | 含义 |
|--------|-----------|------|
| `CEPH_0001` | 404 | 存储池不存在 |
| `CEPH_0002` | 503 | 部分 OSD 离线 |
| `CEPH_0003` | 409 | Bucket 名称已存在 |
| `CEPH_0004` | 503 | 对象存储服务不可用 |

### 7.4 Kubernetes 异常（K8S_xxxx）

| 错误码 | HTTP 状态 | 含义 |
|--------|-----------|------|
| `K8S_0001` | 404 | 命名空间不存在 |
| `K8S_0002` | 404 | 工作负载不存在 |
| `K8S_0003` | 400 | 无法完成扩缩容 |
| `K8S_0004` | 400 | Pod 未在运行状态 |
| `K8S_0005` | 503 | Kubernetes API 不可达 |

### 7.5 任务异常（TASK_xxxx）

| 错误码 | HTTP 状态 | 含义 |
|--------|-----------|------|
| `TASK_0001` | 404 | 任务不存在 |
| `TASK_0002` | 400 | 任务已结束 |
| `TASK_0003` | 500 | 任务执行超时 |
| `TASK_0004` | 409 | 相同资源正在执行其他操作 |
