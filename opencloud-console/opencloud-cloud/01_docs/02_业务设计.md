# opencloud-cloud 业务设计文档

---

## 1. 业务背景

opencloud-cloud 模块是 OpenCloud 云管平台的**云平台对接适配层**，负责封装 OpenStack、Ceph、Kubernetes 三大基础设施平台的 API 调用，为上层提供统一、标准化的资源管理能力。

### 1.1 使用角色

| 角色 | 说明 |
|------|------|
| 超级管理员 | 平台全局权限，管理所有集群配置，不受租户限制 |
| 租户管理员 | 管理所属租户下的云资源，可查看和操作本租户分配的资源 |
| 运维工程师 | 负责云平台运维，执行资源创建、扩容、迁移等操作 |
| 普通用户 | 仅拥有被授权的资源查看权限，可申请资源但无直接操作权限 |

### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| 集群连接管理 | 管理 OpenStack/Ceph/K8s 的连接配置，支持连接测试 |
| 云主机生命周期 | 虚拟机的创建、启动、停止、重启、删除等全生命周期管理 |
| 网络资源管理 | VPC、子网、安全组、浮动 IP 等网络资源的统一管理 |
| 存储资源管理 | 块存储卷、快照、Ceph 存储池、对象存储 Bucket 管理 |
| 容器编排管理 | Kubernetes 工作负载、配置、服务、节点的查看与管理 |
| 资源配额控制 | 创建资源前校验租户配额，防止超用 |
| 状态同步 | 云资源状态定时同步，确保本地数据与云平台一致 |

---

## 2. 业务目标

1. **统一纳管**：提供单一入口管理 OpenStack、Ceph、Kubernetes 三大平台
2. **资源可视化**：实时展示云资源的容量、状态、使用率等关键指标
3. **生命周期管理**：支持云资源的创建、变更、删除等全生命周期操作
4. **配额管控**：实现租户级资源配额管理，防止资源超用
5. **状态同步**：确保控制台数据与底层云平台状态保持一致
6. **安全隔离**：多租户环境下资源隔离，敏感配置加密存储

## 4. 核心业务流程

### 4.1 集群配置管理流程

#### 4.1.1 新增集群配置

**触发条件：** 管理员在集群管理页添加新的云平台连接。

**前置条件：** 管理员具有 `cloud:cluster:add` 权限。

```
管理员提交集群配置信息
  │
  ├─ 参数校验
  │   → name: 必填，租户内唯一，长度 2-64 字符
  │   → type: 必填，枚举 [OPENSTACK, CEPH, KUBERNETES]
  │   → endpoint: 必填，有效的 URL 格式
  │   → 根据 type 校验特定参数
  │
  ├─ 检查集群名称是否已存在（同租户内）
  │   → 存在 → 返回 CLUSTER_NAME_EXISTS
  │
  ├─ 敏感信息加密
  │   → password / secretKey / token 等使用 AES-256-GCM 加密
  │   → 加密密钥来自系统配置 opencloud.encrypt.key
  │
  ├─ 写入数据库 cluster_config
  │   字段: id, tenant_id, name, type, endpoint, config_json(加密), 
  │         status, created_at, updated_at, created_by, updated_by
  │
  ├─ 写入操作日志（operation_log）
  │   字段: userId, tenantId, operationType=CLUSTER_ADD, operateTime
  │
  └─ 返回 200 OK + 集群配置 ID
```

**后置条件：** 集群配置已保存，状态为 ACTIVE，可进行连接测试。

---

#### 4.1.2 集群连接测试

**触发条件：** 管理员对集群配置执行连通性测试。

```
管理员请求 POST /api/cloud/clusters/{id}/test
  │
  ├─ 校验集群配置存在且属于当前租户
  │   → 不存在 → 返回 404 CLUSTER_NOT_FOUND
  │
  ├─ 解密敏感配置信息
  │   → 解密失败 → 返回 500 CONFIG_DECRYPT_ERROR
  │
  ├─ 根据 type 创建对应客户端
  │   ├─ OPENSTACK: 使用 openstack4j 创建 OSClient
  │   │   → 调用 identity().projects().list() 验证连接
  │   ├─ CEPH: 使用 HTTP 客户端访问 Ceph MGR REST API
  │   │   → 调用 /api/health/minimal 验证连接
  │   └─ KUBERNETES: 使用 Fabric8 KubernetesClient
  │       → 调用 apiServer().getVersion() 验证连接
  │
  ├─ 执行连接测试（超时时间 10s）
  │   → 连接成功
  │       → 更新 cluster_config.status = ACTIVE
  │       → 更新 cluster_config.last_check_time = now
  │       → 缓存客户端实例到本地 Map
  │   → 连接失败
  │       → 更新 cluster_config.status = ERROR
  │       → 更新 cluster_config.error_msg = 错误信息
  │
  ├─ 写入操作日志
  │   字段: operationType=CLUSTER_TEST, result=SUCCESS/FAILED, errorMsg
  │
  └─ 返回测试结果
      { success: true/false, message: "连接成功/失败原因", latencyMs: 123 }
```

---

#### 4.1.3 编辑集群配置

**触发条件：** 管理员修改集群连接参数。

```
管理员提交编辑后的集群配置
  │
  ├─ 校验集群配置存在且属于当前租户
  │
  ├─ 参数校验（同新增）
  │
  ├─ 检查名称唯一性（排除自身）
  │
  ├─ 敏感信息处理
  │   → 如提供了新密码 → 重新加密存储
  │   → 如密码字段为空 → 保留原密码不变
  │
  ├─ 更新数据库
  │   → 更新 cluster_config 各字段
  │   → 更新 cluster_config.status = PENDING（需重新测试）
  │
  ├─ 清除该集群的客户端缓存
  │   → 从 clientCache Map 中移除该集群的客户端实例
  │
  ├─ 写入操作日志
  │
  └─ 返回 200 OK
```

**触发条件：** 管理员删除不再使用的集群配置。

```
  ├─ 写入操作日志
  │
  └─ 返回 200 OK
```

### 4.2 OpenStack 云主机管理流程

#### 4.2.1 云主机列表查询

**触发条件：** 用户查看云主机列表页。

```
用户请求 GET /api/cloud/openstack/instances
  │
  ├─ 权限校验（cloud:instance:list）
  │
  ├─ 解析查询参数
  │   → clusterId: 指定集群（可选，默认全部）
  │   → status: 状态筛选（ACTIVE/SHUTOFF/ERROR 等）
  │   → projectId: 项目筛选（可选）
  │   → keyword: 名称搜索（模糊匹配）
  │   → page, size: 分页参数
  │
  ├─ 获取可访问的集群列表
  │   → 超级管理员: 全部集群
  │   → 租户管理员: 本租户配置的集群
  │
  ├─ 并行查询各集群的云主机
  │   ├─ 从缓存获取 OSClient（不存在则创建）
  │   ├─ 调用 Nova API: compute().servers().list()
  │   └─ 转换为本系统统一的 InstanceVO
  │
  ├─ 合并结果并排序
  │
  ├─ 分页处理
  │
  └─ 返回分页结果
      { total: 100, pages: 10, current: 1, records: [...] }
```

#### 4.2.2 创建云主机

**触发条件：** 用户提交创建云主机请求。

**前置条件：** 用户具有 `cloud:instance:create` 权限，配额充足。

```
用户提交创建云主机请求
  │
  ├─ 参数校验
  │   → name: 必填，符合 OpenStack 命名规范
  │   → clusterId: 必填，指定目标集群
  │   → flavorId: 必填，云主机规格
  │   → imageId: 镜像 ID（与 bootFromVolume 二选一）
 │   → networkIds: 至少一个网络
  │   → securityGroupIds: 安全组（可选，使用默认）
  │   → keyPairName: 密钥对（可选）
  │   → bootFromVolume: 是否从卷启动
  │   → volumeSize: 系统盘大小（从卷启动时必填）
  │
  ├─ 配额预检查
  │   → 查询租户当前配额使用情况
  │   → 检查 vCPU: 当前 + flavor.vcpus ≤ quota.vcpu_limit
  │   → 检查内存: 当前 + flavor.ram ≤ quota.memory_limit
  │   → 检查实例数: 当前 + 1 ≤ quota.instance_limit
  │   → 超配额 → 返回 400 QUOTA_EXCEEDED
  │
  ├─ 获取 OSClient 并校验资源存在性
  │   → 校验 flavor 存在且可用
  │   → 校验 image 存在且可用（如指定）
  │   → 校验 network 存在
  │
  ├─ 构建 ServerCreate 对象
  │
  ├─ 调用 Nova API 创建云主机
  │   → server = os.compute().servers().boot(serverCreate)
  │
  ├─ 异步处理（创建为异步操作）
  │   → 立即返回: { instanceId, status: BUILD }
  │   → 后台轮询创建状态
  │
  ├─ WebSocket 推送状态变更
  │   → 创建完成/失败时推送通知
  │
  ├─ 写入操作日志
  │
  └─ 返回 200 OK + 创建任务信息
```

**业务规则：**
- 云主机名称为租户内唯一（同一集群内）
- 创建前必须校验配额，防止超用
- 创建操作异步执行，通过任务 ID 跟踪进度
- 创建失败时自动回滚已分配资源
- 支持从镜像启动或从卷启动（bootFromVolume）
- 系统盘大小：从镜像启动使用镜像最小磁盘要求，从卷启动需指定 volumeSize
- 网络配置：至少指定一个网络，支持多网卡
- 安全组：不指定时使用默认安全组

**创建参数详细说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 是 | 云主机名称，2-255 字符，支持字母数字中文-_. |
| clusterId | String | 是 | 目标集群 ID |
| flavorId | String | 是 | 规格 ID |
| imageId | String | 条件 | 镜像 ID，bootFromVolume=false 时必填 |
| bootFromVolume | Boolean | 否 | 是否从卷启动，默认 false |
| volumeSize | Integer | 条件 | 系统盘大小(GB)，bootFromVolume=true 时必填，≥image.minDisk |
| volumeType | String | 否 | 卷类型，默认集群默认类型 |
| networkIds | List<String> | 是 | 网络 ID 列表，至少一个 |
| securityGroupIds | List<String> | 否 | 安全组 ID 列表 |
| keyPairName | String | 否 | 密钥对名称，用于 SSH 登录 |
| userData | String | 否 | 用户数据脚本（cloud-init），Base64 编码 |
| count | Integer | 否 | 创建数量，默认 1，最大 10 |
| description | String | 否 | 描述信息 |

#### 4.2.3 云主机操作（启动/停止/重启）

**触发条件：** 用户对云主机执行生命周期操作。

```
用户请求 PUT /api/cloud/openstack/instances/{id}/action
请求体: { action: "start" | "stop" | "reboot" | "forceReboot" }
  │
  ├─ 权限校验（cloud:instance:operate）
  │
  ├─ 校验云主机存在且属于当前租户
  │
  ├─ 状态机校验
  │   → start: 仅允许从 SHUTOFF 启动
  │   → stop: 仅允许从 ACTIVE 停止
  │   → reboot: 仅允许从 ACTIVE 重启
  │   → 状态不匹配 → 返回 400 INVALID_STATUS
  │
  ├─ 调用对应 Nova API
  │   → start: server.start()
  │   → stop: server.stop()
  │   → reboot: server.reboot(RebootType.SOFT)
  │   → forceReboot: server.reboot(RebootType.HARD)
  │
  ├─ 异步等待操作完成
  │
  ├─ 更新本地状态缓存
  │
  ├─ 写入操作日志
  │
  └─ 返回 200 OK
```

---

#### 4.2.4 删除云主机

**触发条件：** 用户删除云主机。

```
用户请求 DELETE /api/cloud/openstack/instances/{id}?force={boolean}
  │
  ├─ 权限校验（cloud:instance:delete）
  │
  ├─ 校验云主机存在且属于当前租户
  │
  ├─ 检查是否有关联资源
  │   → 挂载的卷 → 提示先卸载
  │   → 绑定的浮动 IP → 提示先解绑
  │
  ├─ 调用 Nova API 删除
  │   → force=false: 正常删除，需云主机处于 SHUTOFF
  │   → force=true: 强制删除，先停止再删除
  │
  ├─ 异步等待删除完成
  │
  ├─ 释放配额
  │
  ├─ 写入操作日志
  │
  └─ 返回 200 OK
```

### 4.2.6 密钥对（KeyPair）管理

#### 4.2.6.1 密钥对列表

```
GET /api/cloud/openstack/keypairs?clusterId={id}
  │
  ├─ 调用 Nova API: os.compute().keypairs().list()
  │
  ├─ 返回 KeyPairVO 列表
  │   字段: name, fingerprint, publicKey(可选), createdAt
  │
  └─ 返回结果
```

#### 4.2.6.2 创建密钥对

```
POST /api/cloud/openstack/keypairs
{ name, clusterId, publicKey(可选) }
  │
  ├─ 参数校验
  │   → name: 必填，符合 OpenStack 命名规范
  │   → 如提供 publicKey: 导入现有公钥
  │   → 如不提供: 由 OpenStack 生成密钥对
  │
  ├─ 调用 Nova API 创建
  │
  ├─ 如为生成模式，返回私钥（仅一次）
  │
  └─ 返回 200 OK + 密钥对信息（含私钥，如生成）
```

**业务规则：**
- 私钥仅在创建时返回一次，服务端不保存
- 建议用户下载保存私钥

#### 4.2.6.3 删除密钥对

```
DELETE /api/cloud/openstack/keypairs/{name}?clusterId={id}
  │
  ├─ 检查是否有关联云主机（可选）
  │
  ├─ 调用 Nova API 删除
  │
  └─ 返回 200 OK
```

#### 4.9.3 配额检查逻辑

```
创建资源前的配额检查流程
  │
  ├─ 获取租户配额配置
  │   → 无配置 → 默认配额为 0，拒绝创建
  │
  ├─ 获取当前使用量
  │   ├─ OpenStack:
  │   │   → vcpuUsed: 统计所有云主机的 vCPU 总数
  │   │   → memoryUsed: 统计所有云主机的内存总数
  │   │   → instanceUsed: 统计云主机总数

  │   │   → volumeStorageUsed: 统计卷总容量
  │   ├─ Ceph:
  │   │   → storageUsed: 统计本租户所有 Bucket 大小总和
  │   └─ Kubernetes:
  │       → cpuUsed: 统计所有 Deployment 的 requests.cpu
  │       → memoryUsed: 统计所有 Deployment 的 requests.memory
  │
  ├─ 计算新增资源需求
  │
  ├─ 校验
  │   → used + requested > limit → 返回 QUOTA_EXCEEDED
  │   → limit = -1 → 跳过该校验项
  │
  └─ 通过/拒绝
```

## 5. 业务规则

### 5.1 集群配置规则

| 规则 | 说明 |
|------|------|
| 名称唯一性 | 集群名称在同一租户内必须唯一 |
| 敏感信息加密 | 密码、密钥、Token 等使用 AES-256-GCM 加密存储 |
| 连接状态管理 | 集群状态：ACTIVE（正常）、ERROR（连接失败）、PENDING（待测试） |
| 客户端缓存 | 按 clusterId 缓存客户端实例，配置变更时失效缓存 |
| 连接超时 | API 调用超时时间 30s，支持重试 3 次 |

### 5.2 云主机管理规则

| 规则 | 说明 |
|------|------|
| 名称规范 | 符合 OpenStack 命名规范，长度 1-255 字符，不能包含特殊字符 |
| 配额预检 | 创建前必须校验配额，超配额拒绝创建 |
| 异步操作 | 创建、删除、调整规格为异步操作，通过任务跟踪 |
| 状态同步 | 定时同步云主机状态（5 分钟），支持手动刷新 |
| 资源清理 | 删除云主机时自动释放关联的浮动 IP、卷（可选） |

### 5.3 存储管理规则

| 规则 | 说明 |
|------|------|
| 卷状态检查 | 挂载前卷状态必须为 available，卸载前必须为 in-use |
| 扩容限制 | 卷只能扩容不能缩容，扩容需云机关机（取决于后端存储） |
| 快照依赖 | 有快照的卷不能直接删除，需先删除所有快照 |
| 存储配额 | Ceph 存储按租户统计使用量，超配额禁止创建 Bucket |

### 5.4 Kubernetes 管理规则

| 规则 | 说明 |
|------|------|
| 只读优先 | 生产环境建议只读查看，修改操作需二次确认 |
| 敏感信息脱敏 | Secret 的值默认脱敏，不返回明文 |
| 日志流控 | Pod 日志流支持 tail 行数限制，防止大日志拖垮系统 |
| 权限边界 | 通过 RBAC 限制可访问的命名空间 |

### 5.5 配额管理规则

| 规则 | 说明 |
|------|------|
| 默认值 | 新租户默认配额为 0，需管理员手动分配 |
| 无限制标识 | 配额值设为 -1 表示无限制 |
| 实时校验 | 创建资源前实时查询使用量并校验 |
| 告警阈值 | 配额使用率达 80% 时触发告警通知 |

### 5.6 异步任务规则

| 规则 | 说明 |
|------|------|
| 任务持久化 | 所有异步操作创建任务记录，便于跟踪 |
| 超时处理 | 任务执行超时（默认 30 分钟）自动标记为 FAILED |
| 幂等性 | 同一任务的重复提交应返回相同任务 ID |
| 进度更新 | 长耗时任务需定期更新进度字段 |
| 清理策略 | 已完成任务保留 7 天后自动清理 |

### 5.7 状态同步规则

| 规则 | 说明 |
|------|------|
| 定时同步 | 每 5 分钟自动同步一次所有 ACTIVE 集群 |
| 增量优先 | 优先使用云平台 API 的增量查询能力 |
| 失败重试 | 同步失败时重试 3 次，仍失败标记集群 ERROR |
| 缓存策略 | 同步结果缓存 5 分钟，减少 API 调用 |
| 手动刷新 | 支持用户手动触发同步，不受缓存限制 |

## 9. 业务术语表

| 术语 | 英文 | 说明 |
|-----|------|------|
| 集群 | Cluster | 云平台的连接配置，对应一个 OpenStack/Ceph/K8s 实例 |
| 云主机 | Instance | OpenStack Nova 虚拟机实例 |
| 规格 | Flavor | 云主机资源配置模板（vCPU/内存/磁盘） |
| 镜像 | Image | 云主机操作系统模板（Glance 管理） |
| 卷 | Volume | 块存储设备（Cinder 管理） |
| 快照 | Snapshot | 卷的时间点备份 |
| 存储池 | Pool | Ceph 存储资源划分单元 |
| Bucket | Bucket | 对象存储容器（RGW/S3 概念） |
| OSD | Object Storage Daemon | Ceph 对象存储守护进程 |
| PG | Placement Group | Ceph 数据分布单元 |
| 命名空间 | Namespace | Kubernetes 资源隔离单元 |
| Deployment | Deployment | K8s 无状态工作负载 |
| Pod | Pod | K8s 最小部署单元 |
| Service | Service | K8s 服务发现与负载均衡 |
| ConfigMap | ConfigMap | K8s 配置数据存储 |
| Secret | Secret | K8s 敏感数据存储 |
| 配额 | Quota | 租户资源使用限制 |
| 任务 | Task | 异步操作的任务记录 |
| 同步 | Sync | 云资源状态的定时同步操作 |
